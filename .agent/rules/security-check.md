# Security Guidelines

모든 코드는 보안 기준을 준수해야 하며, 커밋 및 배포 전 다음 사항을 반드시 확인합니다.

## 1. 비밀 정보 관리 (Secret Management)

- **Hardcoded Secret 금지**: API Key, 비밀번호, 토큰, 인증 정보 등을 코드 내에 직접 입력하지 마십시오.
- **환경 변수 활용**: 모든 민감 정보는 `.env` 파일과 환경 변수를 통해 관리하십시오.
- **Checklist**: 커밋 전 코드 내에 비밀 정보가 노출되지 않았는지 다시 한 번 스캔합니다.

## 2. 보안 취약점 방지

- **입력값 검증 (Input Validation)**: 모든 사용자 입력값은 신뢰할 수 없는 데이터로 취급하며, 반드시 유효성 검사 및 정화(Sanitization) 과정을 거쳐야 합니다.
- **인젝션 방지**: SQL 쿼리 작성 시 매개변수화된 쿼리(Parameterized Query)를 사용하여 SQL 인젝션을 방지합니다.
- **XSS 예방**: 사용자 입력을 HTML로 렌더링할 경우 반드시 이스케이프 처리를 수행합니다.

## 3. 보안 프로세스

- 보안 관련 이슈를 발견하거나 취약점을 수정할 때는 `/code-reviewer` 워크플로우를 호출하여 다각도로 검토받으십시오.
- 민감한 데이터가 노출된 경우 즉시 관련 정보를 무효화(Rotate)하고 보고하십시오.
