# Testing Requirements

품질 높은 소프트웨어 생성을 위해 다음 테스트 원칙을 준수합니다.

## 1. TDD (Test-Driven Development) 사이클

신규 기능이나 버그 수정 시 에이전트는 다음 워크플로우를 권장합니다.

1. **RED**: 요구사항을 기반으로 실패하는 테스트 코드를 먼저 작성합니다.
2. **GREEN**: 테스트를 통과할 수 있는 최소한의 구현 코드를 작성합니다.
3. **IMPROVE**: 코드를 정리하고 리팩토링하며 성능을 최적화합니다.

## 2. 테스트 커버리지 및 범위

- **최소 커버리지**: 전체 코드의 **80% 이상**이 테스트로 보호되도록 노력합니다.
- **테스크 유형**:
  - **Unit Tests**: 개별 함수 및 로직 검증.
  - **Integration Tests**: 모듈 간 상호작용 및 API 엔드포인트 검증.
  - **E2E Tests**: 핵심 사용자 시나리오 검증.

## 3. 에이전트 활용

- 복잡한 테스트 시나리오 설계 시 `/planner`를 통해 테스트 플랜을 먼저 수립하십시오.
- 테스트 실패 시 원인을 분석하고 구현 코드를 수정하여 해결하십시오. (테스트 코드 자체에 오류가 있지 않은 한 구현 수정을 원칙으로 함)
