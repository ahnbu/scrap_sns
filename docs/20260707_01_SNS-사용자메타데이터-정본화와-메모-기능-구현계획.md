# SNS User Metadata and Notes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not create a worktree unless the user explicitly requests one.

**Goal:** 게시글별 별표, 숨김, 메모를 브라우저 전용 상태에서 파일 정본 기반 사용자 메타데이터로 승격하고, 카드 하단에서 메모를 바로 입력·확인할 수 있게 만든다.

**Architecture:** 서버 응답에 안정적인 `post_key`를 추가하고, 사용자 메타데이터는 `web_viewer/sns_user_metadata.json` 단일 파일을 정본으로 둔다. 브라우저 `localStorage.sns_user_metadata`는 빠른 렌더링과 오프라인 캐시로만 사용하며, 기존 `sns_favorites`, `sns_invisible_posts`는 1회 병합 입력으로 유지한다. `canonical_url`은 사람 확인과 fallback용 보조값으로 보존한다.

**Tech Stack:** Flask `server.py`, Python `utils/post_meta.py`, browser JavaScript `web_viewer/script.js`, `index.html`, CSS, JSON persistence, pytest, Node-based JS unit tests, Playwright UI verification.

## `post_key` 규칙

`post_key`는 아래 순서로 만든다.

1. `normalized_platform + ":" + platform_id`
2. `normalized_platform + ":" + code`
3. `normalized_platform + ":" + urn`
4. `normalized_platform + ":url:" + canonical_url`

플랫폼 정규화:

| 입력 | 출력 |
|---|---|
| `threads`, `thread` | `threads` |
| `linkedin` | `linkedin` |
| `x`, `twitter`, `x/twitter`, `x_twitter` | `x` |
| 기타 | lower-case 원문 |

최신 `output_total/total_full_20260707.json` 기준 사전 점검 결과:

```json
{
  "posts": 1744,
  "urlUnique": 1744,
  "missingUrl": 0,
  "duplicateUrl": 0,
  "platformIdUnique": 1744,
  "duplicatePlatformId": 0
}
```

## 사용자 메타데이터 파일 구조

`web_viewer/sns_user_metadata.json`

```json
{
  "threads:ABC123": {
    "canonical_url": "https://www.threads.com/@user/post/ABC123",
    "favorite": true,
    "hidden": false,
    "note": "강의 사례로 다시 보기",
    "updated_at": "2026-07-07T11:06:00+09:00"
  }
}
```

저장 규칙:

- `favorite`, `hidden`, `note` 중 의미 있는 값이 하나도 없으면 해당 `post_key` 항목을 삭제한다.
- `note`는 trim한 문자열로 저장한다.
- 빈 메모 저장은 note 제거로 처리한다.
- `updated_at`은 KST ISO 문자열로 저장한다.
- 서버 저장은 JSON object만 허용한다.
- 쓰기는 임시 파일에 저장한 뒤 replace하는 atomic write를 사용한다.

## 영구 데이터 영향

| Surface | 영향 |
|---|---|
| `web_viewer/sns_user_metadata.json` | 신규 정본 파일. 별표, 숨김, 메모 저장. |
| `localStorage.sns_user_metadata` | 신규 캐시. 서버 정본 로드 후 동기화. |
| `localStorage.sns_favorites` | 1회 병합 입력으로 읽는다. 삭제하지 않는다. |
| `localStorage.sns_invisible_posts` | 1회 병합 입력으로 읽는다. 삭제하지 않는다. |
| `localStorage.sns_user_metadata_legacy_migrated` | legacy 별표/숨김을 새 구조로 병합했는지 기록한다. |
| `localStorage.sns_folded_posts` | 변경 없음. |
| `web_viewer/sns_tags.json` | 변경 없음. |
| `output_total/total_full_*.json` | 변경 없음. |

기존 데이터 재처리:

- 수집 결과 재생성은 필요 없다.
- 브라우저를 1회 열면 legacy localStorage 상태가 `sns_user_metadata`로 병합되고 서버 파일에 저장된다.
- `localStorage.sns_user_metadata_legacy_migrated === "true"`이면 legacy 별표/숨김을 다시 병합하지 않는다.
- 이 플래그가 없으면 서버 파일을 우선한 뒤 legacy localStorage는 누락 항목 보강에만 사용한다.
- 사용자가 새 metadata 구조에서 별표를 해제하거나 숨김을 복구한 뒤, 이전 `sns_favorites`, `sns_invisible_posts` 때문에 상태가 되살아나면 안 된다.

```python
def normalize_post_key_platform(platform: str) -> str:
    value = str(platform or "").strip().lower()
    if value in {"thread", "threads"}:
        return "threads"
    if value == "linkedin":
        return "linkedin"
    if value in {"x", "twitter", "x/twitter", "x_twitter"}:
        return "x"
    return value


def build_post_key(post: dict) -> str:
    platform = normalize_post_key_platform(post.get("sns_platform"))
    identifier = post.get("platform_id") or post.get("code") or post.get("urn")
    if platform and identifier:
        return f"{platform}:{identifier}"

    canonical_url = canonicalize_url(post)
    if platform and canonical_url:
        return f"{platform}:url:{canonical_url}"
    if canonical_url:
        return f"url:{canonical_url}"
    return ""
```

```javascript
function normalizePostKeyPlatform(platform) {
    const value = String(platform || '').trim().toLowerCase();
    if (value === 'thread' || value === 'threads') return 'threads';
    if (value === 'linkedin') return 'linkedin';
    if (['x', 'twitter', 'x/twitter', 'x_twitter'].includes(value)) return 'x';
    return value;
}

function resolvePostKey(post) {
    if (post?.post_key) return String(post.post_key);
    const platform = normalizePostKeyPlatform(post?.sns_platform);
    const identifier = post?.platform_id || post?.code || post?.urn;
    if (platform && identifier) return `${platform}:${identifier}`;
    const postUrl = resolvePostUrl(post);
    if (platform && postUrl) return `${platform}:url:${postUrl}`;
    return postUrl ? `url:${postUrl}` : '';
}
```

```python
def _atomic_write_json(path, data):
    directory = os.path.dirname(path)
    os.makedirs(directory, exist_ok=True)
    tmp_path = f"{path}.tmp"
    with open(tmp_path, "w", encoding="utf-8") as f:
        json.dump(data, f, indent=4, ensure_ascii=False)
    os.replace(tmp_path, path)


@app.route("/api/get-user-metadata", methods=["GET"])
def get_user_metadata():
    try:
        export_path = os.path.join(WEB_VIEWER_DIR, "sns_user_metadata.json")
        if not os.path.exists(export_path):
            return jsonify({})
        with open(export_path, "r", encoding="utf-8") as f:
            data = json.load(f)
        if not isinstance(data, dict):
            return jsonify({})
        return jsonify(data)
    except Exception:
        logging.exception("Failed to load user metadata")
        return jsonify({"error": "Failed to load user metadata"}), 500


@app.route("/api/save-user-metadata", methods=["POST"])
def save_user_metadata():
    try:
        data = request.get_json(silent=True)
        if data is None:
            return jsonify({"status": "error", "message": "No data received"}), 400
        if not isinstance(data, dict):
            return jsonify({"status": "error", "message": "Invalid data format: expected JSON object"}), 400
        export_path = os.path.join(WEB_VIEWER_DIR, "sns_user_metadata.json")
        _atomic_write_json(export_path, data)
        return jsonify({"status": "success", "message": "User metadata saved successfully"})
    except Exception:
        logging.exception("Failed to save user metadata")
        return jsonify({"status": "error", "message": "Failed to save user metadata"}), 500
```

```python
        """
        const fs = require('fs');
        const src = fs.readFileSync('web_viewer/script.js', 'utf8');
        function extractFunction(name) {
          const patterns = [`function ${name}(`, `async function ${name}(`];
          let start = -1;
          for (const pattern of patterns) {
            start = src.indexOf(pattern);
            if (start !== -1) break;
          }
          if (start === -1) throw new Error(`${name} missing`);
          let depth = 0;
          for (let i = start; i < src.length; i += 1) {

          return post.post_key;

        function resolvePostUrl(post) {
          return post.canonical_url;
        }
        function getKstIsoString() {
```

헬퍼:

```javascript
function pruneUserMetadataEntry(entry) {
    const next = { ...entry };
    if (!next.favorite) delete next.favorite;
    if (!next.hidden) delete next.hidden;
    if (!String(next.note || '').trim()) delete next.note;
    if (!next.favorite && !next.hidden && !next.note) return null;
    return next;
}

function getKstIsoString(now = new Date()) {
    const kst = new Date(now.getTime() + 9 * 60 * 60 * 1000);
    return kst.toISOString().replace('Z', '+09:00');
}

function shouldMergeLegacyUserMetadata() {
    return localStorage.getItem('sns_user_metadata_legacy_migrated') !== 'true';
}

function mergeLegacyStateIntoUserMetadata(metadata, posts, favoriteUrls, hiddenUrls) {
    const byUrl = new Map();
    posts.forEach((post) => {
        const key = resolvePostKey(post);
        const url = resolvePostUrl(post);
        if (key && url) byUrl.set(url, { key, url });
    });

    favoriteUrls.forEach((url) => {
        const target = byUrl.get(url);
        if (!target) return;
        metadata[target.key] = {
            ...(metadata[target.key] || {}),
            canonical_url: target.url,
            favorite: true,
            updated_at: getKstIsoString(),
        };
    });

    hiddenUrls.forEach((url) => {
        const target = byUrl.get(url);
        if (!target) return;
        metadata[target.key] = {
            ...(metadata[target.key] || {}),
            canonical_url: target.url,
            hidden: true,
            updated_at: getKstIsoString(),
        };
    });
}

async function syncUserMetadataToServer() {
    localStorage.setItem('sns_user_metadata', JSON.stringify(userMetadata));
    const response = await fetch('/api/save-user-metadata', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(userMetadata),
    });
    if (!response.ok) throw new Error(`Failed to save user metadata: ${response.status}`);
}
```

```javascript
function renderNoteSection(container, post, options = {}) {
    const postKey = resolvePostKey(post);
    const postUrl = resolvePostUrl(post);
    const entry = userMetadata[postKey] || {};
    const noteText = String(entry.note || '').trim();
    const editing = Boolean(options.editing);

    container.innerHTML = '';
    if (!noteText && !editing) return;

    if (editing) {
        container.innerHTML = `
            <div class="note-editor">
                <textarea class="note-input" rows="3" placeholder="메모 입력...">${escapeHtml(noteText)}</textarea>
                <div class="note-actions">
                    <button class="note-save-btn">저장</button>
                    <button class="note-cancel-btn">취소</button>
                </div>
            </div>
        `;
        const input = container.querySelector('.note-input');
        input.focus();
        container.querySelector('.note-save-btn').addEventListener('click', async () => {
            const value = input.value.trim();
            const nextState = {
                ...(userMetadata[postKey] || {}),
                canonical_url: postUrl,
                note: value,
                updated_at: getKstIsoString(),
            };
            const pruned = pruneUserMetadataEntry(nextState);
            if (pruned) userMetadata[postKey] = pruned;
            else delete userMetadata[postKey];
            await syncUserMetadataToServer();
            renderNoteSection(container, post);
        });
        container.querySelector('.note-cancel-btn').addEventListener('click', () => {
            renderNoteSection(container, post);
        });
        return;
    }

    container.innerHTML = `
        <section class="note-section">
            <div class="note-title">메모</div>
            <p class="note-text">${escapeHtml(noteText)}</p>
        </section>
    `;
}
```

## 완료 기준

- `/api/posts`와 `/api/post/<sequence_id>`가 `post_key`를 반환한다.
- 최신 데이터 기준 `post_key`가 누락 또는 중복 없이 생성된다.
- 별표와 숨김이 `web_viewer/sns_user_metadata.json`에 저장된다.
- 기존 `localStorage.sns_favorites`, `localStorage.sns_invisible_posts`가 새 metadata 구조로 병합된다.
- legacy 병합 완료 후 별표 해제/숨김 복구 상태가 새로고침 뒤에도 되살아나지 않는다.
- `web_viewer/sns_user_metadata.json`은 `.gitignore`로 보호되어 기능 커밋에 실수로 포함되지 않는다.
- 메모가 있는 카드만 하단 `메모` 영역을 표시한다.
- `+note` hover affordance와 입력 흐름이 실제 브라우저에서 동작한다.
- 새로고침 후 메모, 별표, 숨김 상태가 유지된다.
- 관련 unit/integration/UI 검증이 통과한다.
- `README.md`, `docs/development.md`, `CHANGELOG.md`가 새 저장 surface를 반영한다.
