# Threads 로컬 이미지 연결 복구와 재발방지 실행기록

## 발단: 사용자 요청

사용자는 Threads 저장 게시글 중 일부 예전 이미지가 웹 뷰어에서 `Image Unavailable`로 보이는 현상을 보고했다. 사용자는 원래 로컬 이미지로 저장되도록 되어 있었던 것으로 기억했고, 현재 상황 점검과 복구 방안, 향후 재발방지 방안을 요청했다.

확인 결과 이미지 파일 자체는 삭제되지 않았고, 최신 통합 JSON에서 게시글과 로컬 이미지 파일을 연결하는 `local_images` 필드가 유실된 상태였다.

## 작업 상세내역

진단 대상 게시글은 아래 2건이었다.

- `DXDsNvrk7mU`: CHOI, 2026-04-13, Threads
- `DXDcGDykXYz`: 공냥이, 2026-04-13, Threads

조사 결과는 다음과 같다.

| 확인 항목 | 결과 | 근거 |
|-----------|------|------|
| 로컬 이미지 파일 존재 | ✅ 존재 | `web_viewer/images`에 1,718개, 약 249MB 존재 |
| 2026-04-17 통합본 | ✅ 정상 연결 | `DXDsNvrk7mU` 로컬 4개, `DXDcGDykXYz` 로컬 3개 |
| 2026-04-22 최신 통합본 | ❌ 연결 유실 | 두 게시글 모두 `local_images: []` |
| 전체 local_images 추이 | ❌ 급감 | 2026-04-18: 613개 게시글 → 2026-04-22: 0개 게시글 |
| 외부 CDN 상태 | ❌ 만료 | 대상 이미지 URL은 `403`, `wsrv.nl` 프록시는 `404` |

영향받는 영구화 surface는 아래다.

- `output_total/total_full_YYYYMMDD.json`: 최신 통합본의 `local_images` 복구 대상
- `web_viewer/images/`: 기존 로컬 이미지 파일 읽기 대상
- `total_scrap.py`: 앞으로 통합본 생성 시 `local_images` 보존 및 유실 검증 대상
- `tests/contract/`: 재발방지 검증 추가 대상

기존 데이터 마이그레이션은 필요하다. 최신 운영 데이터인 `output_total/total_full_20260422.json`의 `local_images`가 0개이므로, 코드만 고쳐도 현재 웹 뷰어의 `no image` 상태는 복구되지 않는다.

마이그레이션 실행 명령은 아래로 고정한다.

```powershell
node utils/backfill_local_images.mjs --target output_total/total_full_20260422.json --dry-run
node utils/backfill_local_images.mjs --target output_total/total_full_20260422.json --apply
```

마이그레이션 결과 검증 명령과 기대 출력은 아래다.

```powershell
node utils/backfill_local_images.mjs --target output_total/total_full_20260422.json --dry-run
```

기대값: `recoverablePosts`가 0이 아니고, `apply` 후 대상 게시글의 `local_images`가 각각 4개, 3개로 복구된다.

## 의사결정 기록

| 선택지 | 판단 | 이유 |
|--------|------|------|
| 외부 CDN 재시도 | ❌ 제외 | Instagram CDN URL은 만료되어 `403/404`가 발생한다 |
| 이미지 재다운로드 | ⚠️ 보류 | 이미 로컬 파일이 있어 우선순위가 낮고, 만료 URL로는 재다운로드가 불안정하다 |
| 로컬 파일 재연결 | ✅ 채택 | 기존 파일이 있고 `media` URL 해시로 안정적으로 재연결 가능하다 |

- 결정: 기존 `web_viewer/images/<md5(media_url)>.<ext>` 파일을 찾아 최신 통합본의 `local_images`를 backfill한다.
- 근거: 2026-04-17 통합본과 2026-04-22 통합본의 `media` URL 해시가 동일하고, 해당 로컬 파일이 존재한다.
- 트레이드오프: 이미 만료된 외부 URL의 신규 이미지는 복구할 수 없지만, 기존 로컬 파일이 있는 게시글은 즉시 복구할 수 있다.

## 리스크 및 미해결 이슈

- 기존 로컬 파일이 없는 이미지는 이번 backfill로 복구되지 않는다.
- 2026-04-22 이후 생성된 최신 통합본이 추가로 있으면 대상 파일을 최신본으로 바꿔 같은 절차를 적용해야 한다.
- 현재 워킹트리에는 사용자가 만든 것으로 보이는 unrelated 변경이 있어 커밋 범위 분리가 필요하다.
