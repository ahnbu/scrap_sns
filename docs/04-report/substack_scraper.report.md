## 1. 개요

`substack_scrap_by_user.py`를 통해 특정 Substack 사용자의 모든 공개 게시글을 수집하고, 기존 프로젝트의 데이터 관리 규칙에 맞춰 저장하는 기능을 구현하였습니다.

---

## 2. 주요 구현 사항

### 2.1 2단계 수집 로직
- **1단계 (목록 스캔)**: 아카이브 페이지(`list.html` 구조)에서 전체 게시글 URL과 날짜(`datetime`) 정보를 먼저 확보합니다.
- **2단계 (상세 수집)**: 각 게시글 링크를 방문하여 `each_article.html` 구조에서 제목, 부제목, 본문 HTML, 이미지 목록을 정밀하게 추출합니다.

### 2.2 데이터 저장 및 관리
- **저장 경로**: `output_substack/{user_name}/` 하위에 Full 및 Update 데이터를 분리 저장합니다.
- **정렬 및 ID**: 게시글 작성 시간을 기준으로 정렬하고, `sequence_id`를 재할당하여 데이터 일관성을 확보했습니다.
- **보완 로직**: 상세 페이지에서 날짜 정보가 누락된 경우 목록 페이지의 데이터를 활용하는 폴백(Fallback) 로직을 적용했습니다.

### 2.3 안정성 확보
- Playwright의 `networkidle` 대기 및 페이지 간 간격 조정을 통해 안정적인 수집 환경을 구축했습니다.
- UTF-8 인코딩을 준수하여 한글 깨짐 문제를 원천 차단했습니다.
