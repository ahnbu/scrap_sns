# [Report] 저장순 정렬 로직 및 전역 Sequence ID 개선 완료 보고

## 1. 개요
- **작업 명**: 저장순 정렬 로직 개선 및 전역 ID 체계 도입
- **완료 일자**: 2026-02-11
- **목표**: 플랫폼별로 분리되어 있던 ID 체계를 통합하여 웹 뷰어의 "저장순" 정렬 오류 해결

## 2. 주요 변경 사항

### 2.1 데이터 구조 개선
- **`sequence_id`**: 이제 통합 피드 전체를 기준으로 부여되는 **전역 고유 ID**입니다. 수집된 시점이 최신일수록 큰 값을 가집니다.
- **`platform_sequence_id`**: 각 플랫폼(LinkedIn, Threads) 스크래퍼가 부여했던 원래의 ID를 보존합니다. 이를 통해 개별 플랫폼 파일과의 대조가 가능합니다.

### 2.2 통합 로직 고도화 (`total_scrap.py`)
- 기존에 단순히 합치기만 하던 방식에서 탈피하여, 통합 시점에 **ID 재부여 로직**을 추가했습니다.
- 기존 통합 데이터의 최대 ID를 추적하여, 신규 데이터 추가 시 이보다 큰 ID를 연속적으로 부여합니다.
- 신규 데이터 내에서도 날짜순 정렬을 거쳐 ID를 매김으로써 정렬의 일관성을 확보했습니다.

### 2.3 데이터 복구 (Migration)
- 이미 잘못된 ID(1, 2, 3...)로 저장되어 있던 오늘의 데이터 406건에 대해 **전체 재정규화**를 수행했습니다.
- 오늘 스크랩한 링크드인 게시물들이 400번대 ID를 부여받아 웹 뷰어 최상단에 노출되도록 조치했습니다.

## 3. 검증 결과
- **정렬 확인**: 웹 뷰어 실행 시 "저장순" 필터에서 최신 링크드인 및 쓰레드 게시물이 의도한 대로 최상단에 표시됨을 확인했습니다.
- **데이터 추적성**: 모든 게시물에 `platform_sequence_id`가 포함되어 원본 데이터와 대조가 가능함을 확인했습니다.

## 4. 향후 유지보수
- 앞으로 `total_scrap.py`를 실행할 때마다 이 로직이 자동으로 적용되므로, 별도의 수동 작업 없이도 올바른 정렬 순서가 유지됩니다.
