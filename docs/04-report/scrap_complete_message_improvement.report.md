## 2. 주요 조치 사항
### 2.1. 데이터 집계 로직 추가 (`total_scrap.py`)

### 2.2. API 정보 전달 고도화 (`server.py`)
- 스크랩 실행 후 생성된 최신 데이터 파일에서 통계 정보(신규 추가 건수)를 읽어와 UI에 전달하는 기능을 구현하였습니다.

### 2.3. UI 메시지 사용자화 (`web_viewer/script.js`)
- 기존의 정적인 메시지를 대신하여, 서버로부터 전달받은 통계 데이터를 기반으로 한 상세 알림 메시지를 출력합니다.
- **출력 형식**: 
  ```
  총 xx건이 추가되었습니다. 데이터를 새로고침합니다.
  
  쓰레드 - xx건 추가
  링크드인 - xx건 추가
  ```
