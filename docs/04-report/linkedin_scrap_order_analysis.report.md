## 1. 개요

링크드인 사용자 게시글 수집 시, API 응답 순서와 실제 작성 시간이 일치하지 않아 최신글이 중간에 섞이거나 순서가 뒤집히는 문제가 발생했습니다. 이를 해결하기 위해 정렬 로직을 근본적으로 개선했습니다.

---

## 2. 개선 내용

### 2.1 정렬 기준 변경
- **이전**: `sequence_id` (단순 추가 순서) 기준 정렬
- **개선**: `code` (Snowflake ID) 기준 정렬
    - `code`는 생성 시간을 포함하는 고유 ID이므로, 이를 정수(int)로 변환하여 정렬하면 완벽한 시간 순서를 보장할 수 있습니다.

### 2.2 Sequence ID 재할당
- 정렬된 리스트(과거 -> 최신)를 기준으로 `sequence_id`를 1부터 순차적으로 재부여했습니다.
- **결과**: `sequence_id`가 클수록 최신글임을 의미하게 되며, 파일 저장 시에는 최신글(높은 ID)이 상단에 오도록 역순 정렬했습니다.

### 2.3 데이터 병합 로직 강화
- `update_full_version` 메서드에서 기존 데이터와 새 데이터를 병합한 후, 반드시 `code` 기준으로 전체 재정렬을 수행하도록 수정했습니다.
- 이를 통해 수집 시점이나 횟수에 관계없이 항상 일관된 정렬 상태를 유지합니다.
