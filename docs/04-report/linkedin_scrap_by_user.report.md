## 1. 개요

본 프로젝트는 기존의 링크드인 스크래핑 기능을 확장하여, 특정 사용자의 공개된 활동(`recent-activity/all/`) 게시글을 체계적으로 수집하고 관리하는 것을 목표로 하였습니다.

## 2. 주요 구현 내용

### 2.1 핵심 기능

- **사용자별 독립 데이터 관리**: `output_linkedin_user` 하위 폴더 구조를 통해 다수 사용자의 데이터를 섞이지 않게 저장합니다.
- **수집 범위 설정 옵션**: 기간(`--duration`) 및 개수(`--limit`) 인자를 통해 필요한 만큼만 효율적으로 스크랩할 수 있습니다.
- **안정적인 데이터 수집**: Playwright의 네트워크 인터셉션 기능을 활용하여 UI 변경에 강하며, 게시글 본문과 고해상도 이미지를 정확하게 추출합니다.
- **Snowflake ID 기반 날짜 계산**: LinkedIn 고유의 ID 구조를 해석하여 별도의 날짜 텍스트 없이도 정확한 작성 시간을 확보합니다.
