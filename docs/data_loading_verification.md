## 3. 추가 발견 사항

### 🔍 3.1 현재 `script.js`의 로딩 로직 불일치

`index.html`이 `data.js`를 로드하고 있음에도 불구하고, `script.js`는:

- **Line 2**: 하드코딩된 JSON 경로 선언 (`feedJsonPath`)
- **fetchData() 함수**: 이 경로를 `fetch()`로 로드 시도

**결론**: 현재 `data.js`는 로드되지만 **사용되지 않고 있음** (Dead Code).  
실제 데이터 로딩은 `fetch(feedJsonPath)`로 이루어지고 있어, CORS 에러가 발생하면 **로컬 실행이 실패**할 것으로 예상됨.

### 📝 3.2 보고서 보완 필요 사항

검토보고서는 전반적으로 정확하나, 다음 사항을 추가하면 더 완벽해집니다:

1. **현재 로직의 모순 지적**:
   - `index.html`에서 `data.js`를 로드하지만 `script.js`가 이를 활용하지 않음.
   - 실제로는 `fetch()`를 사용하므로 로컬(`file://`) 실행 시 여전히 CORS 에러 발생 가능.

2. **수정 구현 상세**:
   - `script.js`의 `fetchData()` 함수가 다음 순서로 동작하도록 수정 필요:
     1. 서버 API (`/api/latest-data`) 시도
     2. 실패 시 전역 변수 `window.snsFeedData` 확인 및 사용
     3. 둘 다 실패 시 에러 표시
