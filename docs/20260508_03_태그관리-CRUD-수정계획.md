# 태그관리 CRUD 수정 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 태그 관리 화면의 계획 대비 크리티컬 문제만 수정하여, 태그 생성·이름 변경·삭제가 브라우저 기본 팝업이 아니라 화면 안에서 명시적으로 동작하게 만든다.

**Architecture:** 기존 태그 저장 구조와 자동 태그 적용 로직은 유지한다. 수정 범위는 `태그 관리` 탭의 CRUD UI와 그 주변 이벤트 핸들러로 제한한다.

**Tech Stack:** `index.html`, `web_viewer/script.js`, `web_viewer/style.css`, pytest, Playwright smoke test.

---

## 크리티컬 문제

1. CRUD가 화면 안에서 동작하지 않음
   - 계획은 단일 태그 row에서 생성/변경/삭제를 처리하는 구조였음.
   - 실제 구현은 `prompt()`와 `confirm()`에 의존하여 계획 미준수.

2. 중요 액션이 `⋮` 뒤에 숨음
   - 계획의 `[⋯]`는 메뉴 의미였지만 실제 구현은 메뉴가 아니라 텍스트 입력 prompt.
   - 사용자는 작업을 선택하는 것이 아니라 `이름변경` 또는 `삭제`라는 명령어를 입력해야 하는 상태.

3. 태그/키워드 CRUD 초점이 흐려짐
   - `Primary` 노출, 태그명 alias 중복 표시, alias 행 높이 흔들림이 태그 CRUD 목적을 흐림.
   - `Primary`는 화면에서 숨김 유지. 남은 핵심은 CRUD 입력/선택 방식을 화면 안으로 되돌리는 것.

## 목표 UI

기본 row:

```text
gemini (191개)                         [이름 변경] [삭제]
키워드  제미나이  ⊕
```

신규 태그 생성:

```text
[ 태그 검색... ] [+ 태그] [일괄 적용]
↓ + 태그 클릭
[ 새 태그명 입력... ] [저장] [취소]
```

이름 변경:

```text
[ gemini 입력칸 ]                     [저장] [취소]
키워드  제미나이  ⊕
```

삭제 확인:

```text
gemini (191개)                         191개 게시물에서 제거됨 [삭제] [취소]
키워드  제미나이  ⊕
```

키워드 추가:

```text
gemini (191개)                         [이름 변경] [삭제]
키워드  제미나이  [ 새 키워드... ] [저장] [취소]
```

## 완료 기준

- 태그 관리 CRUD 흐름에서 `prompt()`와 `confirm()` 미사용.
- `⋮` 메뉴 미사용.
- `이름 변경`, `삭제`가 row에 명시적으로 노출.
- 신규 태그 생성, 이름 변경, 삭제가 화면 안 inline 상태로 처리.
- `Primary`는 태그 관리 화면에 보이지 않음.
- 데이터 구조와 자동 태그 로직 변경 없음.
