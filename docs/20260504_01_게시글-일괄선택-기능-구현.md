# SNS 뷰어 일괄 선택 기능 구현 실행기록

## 발단: 사용자 요청

SNS 뷰어 카드 헤더의 TODO 버튼을 거의 사용하지 않으므로, 여러 게시글을 선택한 뒤 상단에서 일괄 숨기기, 일괄 별표, 복사를 실행할 수 있는 기능으로 전환하는 방안 검토 및 구현 요청.

## 작업 상세내역

- `feed-my` 검토 결과: TODO 기능보다 게시글 선별·복사 흐름이 실제 사용 가치가 높다는 판단.
- 1차 구현계획 수립: `todo-btn`을 선택 버튼으로 전환하고, 선택된 게시글에 숨기기·별표·복사 적용.
- `plan-check` 검수 결과 반영: 선택 상태를 필터·검색 변경 후에도 유지하는 방식은 화면 밖 게시글까지 작업될 수 있어 위험하므로, 일괄 작업 대상을 현재 화면의 선택 게시글로 제한.
- 검수보고서 저장: `D:/vibe-coding/scrap_sns/docs/plan-check/20260504_bulk-selection-plan_검수보고서.md`

## 의사결정 기록

| 항목 | 현재 화면 선택만 적용 | 필터·검색 간 누적 선택 |
|------|----------------------|------------------------|
| 오작동 방지 | ✅ 화면에 보이는 게시글만 처리 | ❌ 보이지 않는 게시글까지 처리 가능 |
| 사용성 | 명확하고 예측 가능 | 대량 선별에는 유리 |
| 구현 복잡도 | ✅ 낮음 | 상태 설명·해제 UX 필요 |
| 1차 출시 적합성 | ✅ 적합 | ⚠️ 실제 수요 확인 후 확장 권장 |

> 정렬 기준: 실수로 숨기기·별표 처리되는 위험을 최우선으로 판단.

- 결정: 현재 화면에서 선택된 게시글만 일괄 처리.
- 근거: 숨기기와 별표는 사용자의 데이터 선별 상태를 바꾸므로, 화면 밖 선택을 유지하면 의도치 않은 변경 위험이 큼.
- 트레이드오프: 필터를 오가며 누적 선택하는 고급 흐름은 포기하지만, 1차 구현의 예측 가능성과 회귀 위험 감소 확보.

## 실행계획 원문

아래 내용은 linked plan 파일이 아니라 실행 전 TodoList/update_plan을 보존한 것이다.

1. `index.html`에서 TODO 필터 버튼을 제거하고, `globalTagsContainer` 위에 숨김 상태의 `bulkActionBar`를 추가.
2. `web_viewer/script.js`에 `selectedPosts = new Set()`을 추가하되, localStorage에는 저장하지 않음.
3. 선택은 검색어 변경, 필터 변경, 정렬 변경, 데이터 재로드 때 초기화하고, 일괄 작업 대상은 현재 화면에 보이는 선택 게시글로 제한.
4. 기존 `todo-btn`은 `select-btn`으로 변경. 미선택은 `radio_button_unchecked`, 선택은 `check_circle` 아이콘 사용.
5. 기존 TODO 클릭 로직은 제거하되, `sns_todos` 로드와 legacy URL 마이그레이션은 유지.
6. 일괄 액션 헬퍼 추가: `getVisibleSelectedPosts`, `buildBulkCopyText`, `addSelectedUrlsToSet`, `clearSelection`.
7. 일괄 복사는 현재 화면 정렬 순서대로 상세 로드 후 `buildCopyText()` 결과를 `---`로 결합.
8. 일괄 별표는 토글이 아닌 별표 추가만 수행.
9. 일괄 숨기기는 확인창 1회 후 선택 URL 전체를 `sns_invisible_posts`에 저장.

## 리스크 및 미해결 이슈

- 실제 브라우저에서 sticky header와 bulk action bar의 시각적 간섭 가능성. 1차 검증은 단위 테스트 중심으로 수행하고, 필요 시 별도 UI 검증 필요.
- 기존 `sns_todos` 데이터는 유지하지만 UI에서는 더 이상 노출하지 않음. 사용자가 TODO 데이터를 다시 쓰고 싶을 경우 별도 복구 기능 필요.
