# X 인증 오탐 및 갱신 프롬프트 복구 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. 이 레포의 사용자 규칙상 워크트리는 명시 요청이 있을 때만 사용하고, 커밋은 사용자 요청 시 `cp` 스킬로만 수행한다.

**Goal:** X 세션이 유효한데도 업데이트 결과에서 "인증 갱신 필요"가 표시되는 오탐을 제거하고, 실제 인증 갱신이 필요한 경우에만 플랫폼별로 동작 가능한 안내와 검증 명령을 표시한다.

**Architecture:** X producer는 로그인 URL처럼 명시적인 인증 신호가 있을 때만 `SNS_AUTH_REQUIRED`를 낸다. 북마크 페이지에서 신호가 늦거나 Chrome 실행이 흔들리는 경우는 인증 만료가 아니라 수집 실패/불확실 상태로 분리한다. 서버는 Playwright/Chrome을 다시 띄우지 않고, `total_scrap.py`가 보존한 현재 실행의 auth signal payload(`reason`, `current_url`)만으로 X 인증 오탐을 suppress한다.

**Tech Stack:** Python 3.13, Playwright, Flask, Node.js 기반 UI helper 테스트, pytest.

`twitter_scrap.py`의 기존 `should_require_x_auth()` 근처에 다음 함수를 추가하고, 기존 함수는 compatibility wrapper로 유지한다.

```python
def classify_x_auth_state(
    *,
    current_url: str,
    has_tweet_article: bool,
    bookmark_response_seen: bool,
    parsed_bookmark_count: int,
) -> tuple[bool, str]:
    url = (current_url or "").lower()
    if "login" in url or "signup" in url or "challenge" in url:
        return False, "login_required"
    if parsed_bookmark_count > 0:
        return True, "bookmark_response"
    if bookmark_response_seen or has_tweet_article:
        return True, "bookmarks_loaded"
    return False, "no_bookmark_signal"


def should_require_x_auth(
    *,
    current_url: str,
    has_tweet_article: bool,
    bookmark_response_seen: bool,
    parsed_bookmark_count: int,
) -> bool:
    _ready, reason = classify_x_auth_state(
        current_url=current_url,
        has_tweet_article=has_tweet_article,
        bookmark_response_seen=bookmark_response_seen,
        parsed_bookmark_count=parsed_bookmark_count,
    )
    return reason == "login_required"
```

- [x] **Step 3: `twitter_scrap.py`에 transient helper와 producer context launcher를 추가한다**

`WINDOW_*` 설정 아래 또는 `should_require_x_auth()` 근처에 다음 helper를 추가한다.

```python
TRANSIENT_X_BROWSER_ERRORS = (
    "Browser window not found",
    "Target page, context or browser has been closed",
)


def is_transient_x_browser_error(error: Exception) -> bool:
    message = str(error)
    return any(pattern in message for pattern in TRANSIENT_X_BROWSER_ERRORS)


def launch_x_producer_context(playwright, user_data_dir: str):
    last_error = None
    for attempt in range(3):
        try:
            return playwright.chromium.launch_persistent_context(
                user_data_dir=user_data_dir,
                channel="chrome",
                headless=False,
                args=[
                    f"--window-position={WINDOW_X},{WINDOW_Y}",
                    "--disable-blink-features=AutomationControlled",
                ],
                viewport={"width": WINDOW_WIDTH, "height": WINDOW_HEIGHT},
            )
        except Exception as error:
            last_error = error
            if not is_transient_x_browser_error(error) or attempt == 2:
                raise
            time.sleep(1)
    raise last_error
```

```python
def _parse_auth_signal_line(line):
    text = str(line or "").strip()
    prefix = "SNS_AUTH_REQUIRED"
    if not text.startswith(prefix):
        return None
    raw_payload = text[len(prefix):].strip()
    if not raw_payload:
        return None
    try:
        payload = json.loads(raw_payload)
    except json.JSONDecodeError:
        return None
    return payload if isinstance(payload, dict) else None


def _read_auth_signal_from_log(log_path, start_offset=0):
    if not log_path or not os.path.exists(log_path):
        return None
    latest = None
    try:
        with open(log_path, "r", encoding="utf-8", errors="replace") as handle:
            handle.seek(max(0, int(start_offset or 0)))
            for line in handle:
                payload = _parse_auth_signal_line(line)
                if payload:
                    latest = payload
    except OSError:
        return None
    return latest
```

```python
def _is_auth_url(url):
    lowered = str(url or "").lower()
    return any(marker in lowered for marker in ("login", "signup", "challenge"))


def _find_x_auth_signal(x_result):
    if not isinstance(x_result, dict):
        return {}
    signal = x_result.get("auth_signal")
    if isinstance(signal, dict):
        return signal
    for phase in (x_result.get("phases") or {}).values():
        if not isinstance(phase, dict):
            continue
        signal = phase.get("auth_signal")
        if isinstance(signal, dict):
            return signal
    return {}


def _suppress_unreliable_x_auth_required(summary):
    auth_required = list(summary.get("auth_required") or [])
    if "x" not in auth_required:
        return summary

    platform_results = summary.get("platform_results") or {}
    x_result = platform_results.get("x") or {}
    signal = _find_x_auth_signal(x_result)
    if not signal:
        return summary

    reason = str(signal.get("reason") or "").lower()
    current_url = str(signal.get("current_url") or "")
    if reason == "login_required" and _is_auth_url(current_url):
        return summary

    summary["auth_required"] = [platform for platform in auth_required if platform != "x"]
    x_result["status"] = "failed"
    x_result.pop("auth_required", None)
    x_result["auth_suppression"] = {
        "suppressed_auth_required": True,
        "reason": "current_url_not_auth",
        "current_url": current_url,
    }
    platform_results["x"] = x_result
    summary["platform_results"] = platform_results
    return summary
```

## 5. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 과거 log의 stale `SNS_AUTH_REQUIRED`를 현재 실행으로 오인 | 현재 phase 시작 직후 `file_handle.tell()` offset을 저장하고, offset 이후의 signal만 파싱한다 |
| server-side Chrome 재실행으로 `AUTH_HOME/x/user_data` lock 충돌 | 서버에서는 `verify_x_auth.py`, Playwright, Chrome을 실행하지 않는다 |
| X producer가 no signal 상태로 계속 진행해 0건 업데이트가 될 수 있다 | auth_required 대신 failed/0건 경로로 분리하고 로그에 "북마크 로딩 신호 미확인"을 남긴다 |
| 실제 로그인 만료인데 X가 bookmark URL을 유지하는 예외 상황 | producer가 `login/signup/challenge` URL만 인증필요로 판정하고, final 검증에서 `verify_x_auth.py`를 별도로 실행한다 |
| 테스트에서 실제 auth 파일을 건드릴 수 있다 | unit/integration tests는 mock과 log fixture만 사용하고, 실제 auth 검증은 최종 수동 단계로 분리한다 |

### 6.2 회귀 및 critical race 재점검

| 리스크 | 판정 | 계획 반영 |
|---|---|---|
| `AUTH_HOME/x/user_data` 동시 launch race | ✅ 해소 | 서버 재검증을 제거하고 producer만 실행 중 profile을 연다 |
| stale auth signal race | ⚠️ 테스트 필요 | `log_offset` 기반 parser와 stale signal 무시 테스트 추가 |
| auth signal 누락 시 오탐 suppress | ✅ 방지 | signal이 없으면 기존 `auth_required`를 유지한다 |
| 실제 auth URL suppress 회귀 | ✅ 방지 | login URL일 때 `auth_required` 유지 테스트 추가 |
| UI에서 failed와 auth_required 혼동 | ✅ 방지 | `auth_suppression` 결과가 auth notice가 아니라 failed notice에만 들어가는 테스트 추가 |
| 서버 응답 지연 회귀 | ✅ 방지 | server request 중 Playwright/Chrome/subprocess 재검증 없음 |

## 7. 부록 A: 문제 체인

1. 사용자가 앱에서 업데이트 실행.
2. `server.py:/api/run-scrap`이 `total_scrap.py --mode update` 실행.

5. 기존 구현은 3초 내 article/network 신호가 없으면 `should_require_x_auth()`가 `True`를 반환했다.
6. orchestrated run이므로 `SNS_AUTH_REQUIRED {"platform":"x","reason":"login_required"}` 출력 후 exit code 86이 발생했다.
7. 보완 후 `total_scrap.py`는 현재 phase log offset 이후의 `SNS_AUTH_REQUIRED` payload만 `auth_signal`로 보존한다.
8. 보완 후 `server.py`는 X `auth_signal.current_url`이 `login/signup/challenge`가 아니면 `auth_required`를 suppress하고 `failed`로 분리한다.

10. 보완 전 프롬프트 검증 명령은 X가 아니라 LinkedIn으로 고정되어 있었다.
