﻿# SNS 세션 인증정보(Auth) 갱신 가이드

이 문서는 스크래퍼 운영을 위해 필요한 SNS 세션 인증정보(uth/*.json, uth/x_user_data)를 갱신하는 방법을 설명합니다. SNS의 보안 정책(봇 탐지)으로 인해 자동 로그인이 막힐 경우, 이 가이드를 따라 수동으로 세션을 갱신해야 합니다.

## 1. 개요 (Session Management)

| SNS 플랫폼 | 저장 방식 | 저장 경로 | 사용 스크립트 |
|------------|-----------|-----------|---------------|
| **Threads** | Playwright JSON | uth/auth_threads.json | 	hread_scrap.py, 	hread_scrap_single.py |
| **LinkedIn** | Playwright JSON | uth/auth_linkedin.json | linkedin_scrap.py |
| **Twitter(X)** | Persistent Context | uth/x_user_data/ | 	witter_scrap.py, 	witter_scrap_single.py |

---

## 2. 세션 갱신 방법 (Manual Renewal)

### 🔹 Threads & LinkedIn 갱신
Threads와 LinkedIn은 간단한 JSON 세션 파일을 사용합니다.

1. 터미널에서 아래 명령어를 실행합니다:
   `powershell
   python renew_auth.py
   `
2. 메뉴에서 원하는 번호를 선택합니다 (1: LinkedIn, 2: Threads).
3. 브라우저 창이 뜨면 **직접 로그인을 완료**합니다.
4. 로그인이 완료되어 피드가 보이면, 터미널로 돌아와 y를 입력합니다.
5. uth/ 폴더에 세션 파일이 업데이트됩니다.

### 🔹 Twitter(X) 갱신
Twitter는 봇 탐지가 강력하여 실제 크롬 브라우저의 프로필 데이터(Persistent Context)를 사용합니다.

1. **중요**: 현재 열려 있는 모든 Google Chrome 브라우저 창을 닫습니다.
2. 터미널에서 아래 명령어를 실행합니다:
   `powershell
   python renew_twitter_auth.py
   `
3. 실제 크롬 브라우저 창이 뜨면 로그인을 진행합니다. (이미 로그인되어 있을 수도 있습니다)
4. 로그인이 완료되면 터미널에서 y를 입력하여 종료합니다.
5. uth/x_user_data/ 내의 프로필 데이터가 최신 상태로 유지됩니다.

## 3. 트러블슈팅 (Troubleshooting)

### ⚠️ "Could not log you in now" (Twitter)
- **원인**: Twitter의 봇 탐지 시스템이 자동화 도구를 감지한 경우입니다.
- **해결**: 
  - enew_twitter_auth.py를 사용하여 실제 Chrome을 통해 접속하세요.
  - 그래도 실패할 경우, 스마트폰 테더링(핫스팟)을 사용하여 IP 주소를 변경한 후 시도하세요.

### ⚠️ "502 Bad Gateway" (Cloudflare)
- **원인**: 단시간에 너무 많은 요청을 보내 서버에서 일시적으로 차단한 경우입니다.
- **해결**: 약 10~15분 정도 대기한 후 다시 시도하세요.

### ⚠️ "Executable doesn't exist" (Playwright)
- **원인**: 브라우저 바이너리가 설치되지 않았거나 경로가 어긋난 경우입니다.
- **해결**: playwright install chromium 또는 python -m playwright install chromium을 실행하세요.
