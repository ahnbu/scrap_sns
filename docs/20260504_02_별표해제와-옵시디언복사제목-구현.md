# SNS 뷰어 별표 해제와 Obsidian 복사 제목 구현 실행기록

## 발단: 사용자 요청

직전 커밋(`d6c6448`)으로 추가된 게시글 다중 선택 기능에서 별표는 일괄 추가만 가능하므로, 이미 별표된 게시글을 일괄 해제하는 기능 필요. 또한 여러 게시글을 일괄 복사할 때 Obsidian Markdown 붙여넣기 후 게시글별 제목으로 분리되도록 각 게시글 블록 첫 줄에 `##` heading 추가 요청.

## 작업 상세내역

- 작업 범위는 사용자 지시에 따라 직전 커밋 이후 변경 2건으로 제한.
- 대상 1: bulk action bar에 `별표 해제` 버튼 추가.
- 대상 2: 일괄 복사 결과에서 각 게시글 블록 앞에 `## {첫 번째 유효 본문 줄}` 추가.
- 개별 게시글 복사 포맷은 유지. 일괄 복사 전용 포맷만 변경.

## 의사결정 기록

| 항목 | 채택안 | 제외안 |
|------|--------|--------|
| 별표 해제 UX | ✅ `별표 추가`와 `별표 해제` 버튼 분리 | ❌ 단일 토글 버튼 |
| 일괄 복사 heading | ✅ bulk copy에만 `##` 추가 | ❌ 개별 복사까지 포맷 변경 |
| 제목 추출 방식 | ✅ 첫 번째 유효 본문 줄, 없으면 작성자 fallback | 본문 전체 또는 고정 제목 |

> 정렬 기준: mixed selection에서 의도치 않은 별표 제거 방지와 기존 개별 복사 회귀 최소화를 우선.

- 결정: 별표 추가/해제는 별도 버튼으로 분리.
- 근거: 선택 게시글 중 일부만 별표된 상태에서 토글은 결과 예측이 어려움.
- 트레이드오프: 버튼이 하나 늘지만, 사용자가 실행 결과를 더 명확하게 예측 가능.

## 실행계획 원문

아래 내용은 linked plan 파일이 아니라 실행 전 TodoList/update_plan을 보존한 것이다.

1. `index.html` bulk action bar에 `bulkUnfavoriteBtn` 버튼 추가.
2. `web_viewer/script.js`에 `bulkUnfavoriteBtn` DOM 참조와 `removeSelectedUrlsFromSet()` 헬퍼 추가.
3. 선택된 현재 화면 게시글 URL만 `favorites.delete(url)`로 제거하고 `sns_favorites`에 저장.
4. `getBulkCopyHeading(post)` 헬퍼 추가.
5. `buildBulkCopyText(posts)`를 `## {heading}\n\n{buildCopyText(post)}` 조합으로 변경.
6. `tests/unit/test_web_viewer_bulk_selection.py`에 별표 해제와 bulk heading 테스트 추가.

## 리스크 및 미해결 이슈

- `##` heading은 본문 첫 줄이 길 경우 Obsidian outline에 긴 제목으로 표시될 수 있음. 1차 구현에서는 80자로 제한.
- 본문 첫 줄이 이미 Markdown heading이면 중복 heading 방지를 위해 앞의 `#` 접두를 제거.
