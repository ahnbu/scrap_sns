# 워크플로우 가이드 (development.md 끝에 추가할 내용)

## 13. 스크랩 및 병합 워크플로우 가이드 (2026-02-06)

### 1. 워크플로우 구분

프로젝트에는 두 가지 주요 워크플로우가 있습니다:

#### A. 정기 수집 워크플로우 (전체 프로세스)

**스크립트**: `total_scrap.py`

**동작 방식**:

1. 모든 스크래퍼를 순차 실행 (Threads, LinkedIn)
2. 각 플랫폼의 최신 데이터 수집
3. 수집된 데이터를 자동으로 병합
4. `output_total/total_full_YYYYMMDD.json` 및 `web_viewer/data.js` 생성

**사용 시나리오**:

- 정기적인 데이터 업데이트
- 신규 게시물 자동 수집
- 전체 프로세스를 한 번에 실행하고 싶을 때

**실행 명령**:

```bash
# 증분 업데이트 (기본값)
python total_scrap.py --mode update

# 전체 재수집
python total_scrap.py --mode all
```

**특징**:

- ✅ 자동화된 전체 파이프라인
- ✅ 스크래핑부터 병합까지 모두 처리
- ⚠️ 시간이 오래 걸림 (수 분)
- ⚠️ 불필요한 재수집 발생 가능

#### B. 병합 전용 워크플로우 (빠른 업데이트)

**스크립트**: `merge_and_update.py`

**동작 방식**:

1. 기존 수집된 파일만 읽기 (스크래핑 없음)
2. 데이터 병합
3. `output_total/total_full_YYYYMMDD.json` 및 `web_viewer/data.js` 생성

**사용 시나리오**:

- 데이터는 이미 수집되었지만 병합만 다시 하고 싶을 때
- 수동으로 데이터 파일을 수정한 후 웹 뷰어 업데이트
- 개행 문자 보존 등의 데이터 수정 작업 후 재병합

**실행 명령**:

```bash
python merge_and_update.py
```

**특징**:

- ✅ 매우 빠름 (수 초)
- ✅ 스크래핑 없이 병합만 수행
- ✅ 데이터 수정 작업 후 즉시 반영
- ⚠️ 새로운 게시물 수집은 하지 않음

### 2. 워크플로우 선택 가이드

| 상황                     | 권장 스크립트                  | 이유                   |
| ------------------------ | ------------------------------ | ---------------------- |
| 새로운 게시물 수집 필요  | `total_scrap.py`               | 스크래핑 + 병합 자동화 |
| 데이터 파일 수동 수정 후 | `merge_and_update.py`          | 빠른 재병합            |
| 정기 스케줄 실행         | `total_scrap.py --mode update` | 자동화된 증분 업데이트 |
| 개행 등 텍스트 수정 후   | `merge_and_update.py`          | 수정사항 즉시 반영     |
| 웹 뷰어만 업데이트       | `merge_and_update.py`          | 불필요한 수집 생략     |

### 3. 주의사항

**`total_scrap.py`의 동작 방식:**

- `run_scrapers()` 함수가 **무조건 모든 스크래퍼를 실행**하도록 구현되어 있습니다.
- 이미 최신 데이터가 있어도 재수집을 시도하므로, 단순히 병합만 필요한 경우에는 비효율적입니다.

**해결 방법:**

- 병합만 필요한 경우: `merge_and_update.py` 사용
- 정기 수집: `total_scrap.py` 사용

### 4. 관련 스크립트

| 스크립트              | 용도                 | 실행 시점                |
| --------------------- | -------------------- | ------------------------ |
| `threads_scrap.py`    | Threads 데이터 수집  | 개별 플랫폼 수집 필요 시 |
| `linkedin_scrap.py`   | LinkedIn 데이터 수집 | 개별 플랫폼 수집 필요 시 |
| `total_scrap.py`      | 전체 수집 + 병합     | 정기 자동화 실행         |
| `merge_and_update.py` | 병합 전용            | 데이터 수정 후 빠른 반영 |
