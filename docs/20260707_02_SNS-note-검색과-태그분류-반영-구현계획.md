# SNS Note Search and Tagging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not create a worktree unless the user explicitly requests one. Use the `cp` skill for commits; do not run raw `git add` or `git commit`.

**Goal:** `web_viewer/sns_user_metadata.json`에 저장된 `note`를 웹 검색, 자동 태그 분류, `utils/query-sns.mjs` CLI 검색과 출력에 반영한다.

**Architecture:** `output_total/total_full_*.json`은 계속 수집 데이터 정본으로 두고, 사용자 메타데이터는 `post_key`로 런타임에 join한다. 서버는 note를 내부 검색 인덱스와 자동 태그 haystack에만 포함하고, 브라우저는 기존 `/api/get-user-metadata`를 통해 note를 표시한다. CLI는 같은 metadata 파일을 읽어 검색 대상과 결과 출력에 note를 포함한다.

**Tech Stack:** Flask `server.py`, Python `utils/post_meta.py`, browser JavaScript `web_viewer/script.js`, Node CLI `utils/query-sns.mjs`, pytest integration tests, Node CLI verification test, Playwright UI verification.

---

## 현재 진단

- `web_viewer/sns_user_metadata.json`에는 `note`가 저장된다.
- 서버 검색 인덱스는 `server.py`의 `_load_latest_posts()`에서 `full_text`, `display_name`, `username`만 `_searchable`에 넣는다.
- 서버 자동 태그 분류는 `/api/auto-tag/apply`에서 `full_text`, `display_name`, `username`만 `haystack`에 넣는다.
- `utils/query-sns.mjs search`는 `full_text`, `display_name`, `username`, `tags`만 검색한다.
- 2026-07-07 KST 확인 기준, note 안에만 있는 `sinmb79`는 `/api/search`, `/api/auto-tag/apply`, `node utils/query-sns.mjs search sinmb79` 모두 0건이다.

## 범위

포함:

- 웹 검색에서 note 텍스트가 검색된다.
- 웹 자동 태그 분류에서 note 텍스트가 태그 규칙 입력으로 사용된다.
- note 변경 후 자동 태그 재실행 조건이 갱신된다.
- `utils/query-sns.mjs` 검색에서 note 텍스트가 검색된다.
- CLI JSON, brief, Markdown export에서 note가 확인 가능하다.
- 테스트와 실제 로컬 데이터 smoke 검증이 추가된다.

제외:

- `output_total/total_full_*.json`에 note를 쓰지 않는다.
- `web_viewer/sns_tags.json` key 구조를 바꾸지 않는다.
- note 기반 태그와 본문 기반 태그를 별도 출처 필드로 영구 저장하지 않는다.
- note 전문을 `/api/posts` 목록 응답에 새 필드로 노출하지 않는다.

## 영구 데이터 영향

영향받는 저장 대상:

- `web_viewer/sns_user_metadata.json`: 기존 구조 유지. `note` 조회 범위만 넓어진다.
- `web_viewer/sns_tags.json`: 자동 태그 재실행 시 note 기반 매칭 태그가 추가될 수 있다.
- `localStorage.sns_auto_tag_last_signature`: note 변경이 자동 태그 재실행 조건에 포함되도록 값 의미가 바뀐다.

기존 데이터 재처리:

- 수집 JSON 재생성은 필요 없다.
- 기존 `sns_user_metadata.json`은 그대로 사용한다.
- 기존 `sns_tags.json`은 자동 태그를 다시 실행할 때만 보강된다. 기존 태그 삭제는 하지 않는다.

def _get_file_state(path):
    if not os.path.exists(path):
        return {"path": path, "mtime": None, "size": None}
    stat = os.stat(path)
    return {"path": path, "mtime": stat.st_mtime_ns, "size": stat.st_size}


def _load_user_metadata():
    path = _get_user_metadata_path()

function buildPostKey(post) {
  const platform = normalizePlatformName(post.sns_platform);
  const identifier = post.platform_id || post.code || post.urn;
  if (platform && identifier) {
    return `${platform}:${identifier}`;
  }

  const url = resolvePostUrl(post);
