# linkedin_user_data_merge 설계서

> **요약**: 다수의 JSON 파일을 읽어 `code` 필드 기반으로 중복을 제거하고 단일 파일로 병합하는 로직을 설계합니다.
>
> **프로젝트**: scrap_sns
> **버전**: 1.0.0
> **작성자**: Gemini CLI
> **날짜**: 2026-02-07
> **상태**: Draft
> **Planning Doc**: [linkedin_user_data_merge.plan.md](../01-plan/features/linkedin_user_data_merge.plan.md)

---

## 1. 개요

### 1.1 설계 목표

- 여러 경로에 분산된 JSON 데이터를 손실 없이 수집.
- 게시물 고유 ID인 `code`를 사용하여 완벽한 중복 제거 수행.
- 병합 후의 메타데이터(총 건수, 병합 일시 등)를 갱신하여 저장.

### 1.2 설계 원칙

- **유연성**: 파일 구조가 리스트 형태이든 메타데이터를 포함한 딕셔너리 형태이든 유연하게 처리.
- **가독성**: 작업 결과를 직관적인 요약 리포트로 제공.

---

## 2. 구현 상세

### 2.1 대상 파일 및 경로

1.  **Main File**: `output_linkedin_user/gb-jeong/python/linkedin_py_full_20260207.json`
2.  **Temp Directory**: `output_linkedin_user/temp/` (모든 `.json` 파일)

### 2.2 로직 흐름

1.  **데이터 수집**:
    - 모든 대상 파일을 순회하며 로드.
    - 파일 내 `posts` 리스트(딕셔너리 구조인 경우) 또는 리스트 자체를 추출.
2.  **중복 제거**:
    - `dict` 자료구조 활용: `code`를 키(key)로, 데이터 객체를 값(value)으로 저장.
    - 나중에 읽은 파일의 데이터가 기존 데이터를 덮어쓰도록 하여 최신성 유지 (필요 시).
3.  **데이터 정렬**:
    - `created_at` 또는 `code`를 기준으로 내림차순 정렬 (최신글 우선).
4.  **시퀀스 재할당**:
    - 병합된 전체 데이터에 대해 `sequence_id`를 1부터 다시 부여.
5.  **저장**:
    - `linkedin_py_full_20260207_total.json` 이름으로 저장.

### 2.3 요약 정보 항목

- 파일별 로드된 게시물 수
- 중복으로 제거된 게시물 총 수
- 최종 통합된 게시물 수
- 저장 경로 및 파일명
