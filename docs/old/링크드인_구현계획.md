# [구현 계획] 링크드인 저장된 게시물 수집기 (JS/Playwright 버전)

## 📊 현황 진단 리포트 (2026-01-30)

### 🔴 문제 상황

현재 `linkedin_scrap.js`가 **데이터 수집 실패 (0건)** 상태입니다. 디버그 파일 분석 결과:

- ✅ **세션 유효**: 프로필 데이터 로드 확인 (로그인 상태 정상)
- ❌ **API 누락**: 저장된 게시물 API 요청이 캡처되지 않음
- ❌ **HTML 빈 화면**: 게시물 렌더링 전 스크립트 종료

### 🔍 근본 원인 분석

#### 1. **URL 필터링 오류**

**현재 코드 (잘못된 패턴)**:

```javascript
if (url.includes('graphql') || url.includes('api') || json.data || json.included)
```

**실제 API 엔드포인트**:

```
https://www.linkedin.com/voyager/api/graphql?variables=(...)&queryId=voyagerSearchDashClusters.ef3d0937fb65bd7812e32e5a85028e79
```

**문제점**: 너무 광범위한 필터로 인해 관련 없는 요청까지 모두 캡처하면서 정작 핵심 엔드포인트를 정확히 식별하지 못함.

#### 2. **데이터 파싱 로직 완전 오류**

**현재 코드**:

```javascript
if (json.data && json.data.searchDashClustersByAll) {
  entities = json.data.searchDashClustersByAll.elements || [];
}
```

**실제 응답 구조** (response.json 분석):

```json
{
  "data": {
    "data": {  // ⚠️ 이중 중첩!
      "searchDashClustersByAll": {
        "elements": [
          {
            "items": [
              {
                "*entityResult": "urn:li:fsd_entityResultViewModel:(...)"
              }
            ]
          }
        ]
      }
    }
  },
  "included": [  // ⚠️ 실제 게시물 내용이 여기에!
    {
      "$type": "com.linkedin.voyager.dash.search.EntityResultViewModel",
      "trackingUrn": "urn:li:activity:7419563139156992000",
      "title": { "text": "SangRok Jung" },  // 작성자
      "primarySubtitle": { "text": "AI 자동화 전문가 | ..." },  // 직책
      "summary": { "text": "웹디자인, 이제 AI한테..." },  // 게시물 본문
      "secondarySubtitle": { "text": "1주 • " },  // 시간
      "image": { ... },  // 프로필 이미지
      "entityEmbeddedObject": { "image": {...} }  // 첨부 이미지
    }
  ]
}
```

**핵심 발견**:

- `data.data` 이중 중첩 구조
- 게시물 목록은 `elements[0].items[]`에 URN만 존재
- 실제 내용은 `included` 배열의 `EntityResultViewModel` 타입에 저장
- URN을 키로 매핑 필요

#### 3. **타이밍 이슈**

- 페이지 로드 직후 API 요청이 발생하지 않음
- 스크롤 동작 전에 스크립트가 "데이터 없음"으로 판단

### 2. 데이터 파싱 로직 (3단계)

#### Step 1: URN 목록 추출

```javascript
const urnList =
  json?.data?.data?.searchDashClustersByAll?.elements?.[0]?.items || [];
// urnList = [{ "*entityResult": "urn:li:fsd_entityResultViewModel:(...)" }, ...]
```

#### Step 2: 상세 데이터 매핑 (included 배열)

```javascript
const entityMap = new Map();
(json.included || []).forEach((item) => {
  if (item.$type === "com.linkedin.voyager.dash.search.EntityResultViewModel") {
    const activityUrn = item.trackingUrn; // "urn:li:activity:7419563139156992000"
    entityMap.set(item.entityUrn, item); // entityUrn을 키로 저장
  }
});
```

#### Step 3: 병합 및 추출

```javascript
urnList.forEach(({ "*entityResult": urn }) => {
  const entity = entityMap.get(urn);
  if (!entity) return;

  const post = {
    code: entity.trackingUrn, // "urn:li:activity:..."
    username: entity.title?.text || "Unknown",
    full_text: entity.summary?.text || "",
    time_text: entity.secondarySubtitle?.text?.replace(" • ", "").trim() || "",
    post_url:
      entity.navigationUrl ||
      `https://www.linkedin.com/feed/update/${entity.trackingUrn}`,
    images: extractImages(entity),
    created_at: new Date().toISOString().replace("T", " ").split(".")[0],
    source: "packet",
  };

  collectedItems.set(post.code, post);
});
```

### 3. 이미지 추출 헬퍼 함수

```javascript
function extractImages(entity) {
  const images = [];

  // 첨부 이미지 (entityEmbeddedObject)
  const embeddedImg =
    entity.entityEmbeddedObject?.image?.attributes?.[0]?.detailData
      ?.vectorImage;
  if (embeddedImg?.artifacts?.length > 0) {
    const largestArtifact = embeddedImg.artifacts.sort(
      (a, b) => b.width - a.width,
    )[0];
    const imgUrl =
      embeddedImg.rootUrl + largestArtifact.fileIdentifyingUrlPathSegment;
    images.push(imgUrl);
  }

  return images.filter(Boolean);
}
```
