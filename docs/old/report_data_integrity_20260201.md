### 2.1 상세 현황

- **LinkedIn**: 수집된 **모든** 게시물에서 작성 날짜 정보가 없습니다. 현재 웹 뷰어에서는 전량 '수집일(Crawled Date)'로 대체 표시되고 있어, '최신순 정렬' 기능이 LinkedIn 게시물에는 사실상 작동하지 않는 상태입니다.
- **Threads**: 약 1/4의 게시물에서 날짜 파싱에 실패했습니다. 실패한 게시물들은 특정 패턴이나 구조를 가질 가능성이 높습니다.

## 3. 원인 추정

1.  **LinkedIn 스크래퍼 (`linkedin_scraper.py`)**
    - 게시물 내 날짜 정보를 담고 있는 DOM 요소(예: `.time`, `span.visually-hidden` 등)를 정확히 타겟팅하지 못하고 있습니다.
    - LinkedIn은 날짜를 "2일 전", "3시간 전"과 같은 상대적 텍스트로 표시하는 경우가 많은데, 이를 파싱 하거나 절대 날짜로 변환하는 로직이 누락되었거나 작동하지 않습니다.

2.  **Threads 스크래퍼**
    - 스크롤 로딩 중 동적으로 변경되는 DOM 구조나, 특정 유형의 게시물(예: 답글, 리포스트 등)에서 날짜 위치가 달라져 파싱에 실패했을 가능성이 있습니다.

## 4. 개선 계획 (Action Plan)

### 4.1 단기 조치 (웹 뷰어 - 완료)

- **UI 표시**: 작성일(`created_at`)이 없는 경우, **수집일(`crawled_at`)**을 대신 표시하되 `(Crawled)`라는 접두어를 붙여 사용자가 인지할 수 있도록 조치했습니다.
- **정렬 로직**: 정렬 시 `created_at`이 없으면 `crawled_at`을 대체 정렬 키로 사용하여 정렬 기능이 깨지지 않도록 방어 로직을 추가했습니다.
