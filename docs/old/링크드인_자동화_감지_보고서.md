# [보고서] 링크드인 자동화 도구 감지 및 콘텐츠 제한 분석

## 1. 개요

최근 링크드인(LinkedIn) 스크래퍼 개발 과정에서 `Playwright`와 `MCP(Model Context Protocol)`를 사용할 때, 특정 콘텐츠(특히 '저장된 게시물')가 정상적으로 로드되지 않거나 일부만 표시되는 현상이 발생하였습니다. 이에 대한 원인 분석과 실험 결과를 정리합니다.

## 2. 주요 현상 (Symptoms)

- **콘텐츠 제한**: 정상적인 브라우징 시에는 수십 개의 저장된 게시물이 보이나, 자동화 도구 연결 시 초기 10개 항목만 표시되거나 "결과 더보기" 버튼이 작동하지 않음.
- **API 응답 이상**: `voyagerSearchDashClusters` API 요청 시 데이터가 누락되거나, `NullValueInNonNullableField`와 같은 GraphQL 에러가 발생.
- **감지 징후**: 페이지 로딩 속도가 비정상적으로 느려지거나, 특정 인터랙션(스크롤, 버튼 클릭)이 무시됨.

## 3. 원인 분석 (Root Cause)

### 3.1 LinkedIn Anti-Bot 시스템

LinkedIn은 업계 최고 수준의 봇 감지 시스템을 운용하며, 다음과 같은 지문(Fingerprint)을 통해 자동화 도구를 식별합니다.

- **Playwright 특성 감지**: CDP(Chrome DevTools Protocol) 연결 시 발생하는 특정 지연 및 프로빙 신호.
- **확장 프로그램 간섭**: `Playwriter`와 같은 도구가 브라우저 지문을 변조하거나 자동화 도구임을 노출하는 경우.
- **비정상적 사용자 행동**: 인간의 마우스 이동, 스크롤 속도와 다른 일정한 패턴의 명령 수행.

### 3.2 개인화 데이터 보호 정책

LinkedIn은 '저장된 게시물'을 매우 민감한 개인 데이터로 간주하여, 자동화 의심이 있는 "저품질 사용자" 세션에 대해서는 데이터 노출 범위를 대폭 제한합니다 (Rate Limiting 및 Content Throttling).

## 4. 해결 및 우회 방안 방안 (Proposed Solutions)

### 4.1 스텔스(Stealth) 기술 적용

- **Playwright-Stealth**: WebGL, Canvas, User-Agent 지문을 랜덤화하여 90% 이상의 자동화 감지 우회 가능.
- **Custom User-Agent**: 실제 사용 중인 브라우저의 `chrome://version/` 정보와 완벽히 일치하는 User-Agent 설정.

### 4.2 휴먼라이크(Human-like) 인터랙션

- **랜덤 지연**: `page.waitForTimeout(random(2000, 5000))`과 같이 일정한 패턴이 없는 대기 시간 삽입.
- **점진적 스크롤**: 한 번에 큰 폭으로 스크롤하지 않고 실제 인간이 읽는 속도로 스크롤 유도.

### 4.3 프로필 및 세션 관리

- **별도 프로필 사용**: 자동화 전용 Chrome 프로필을 생성하여 수동 로그인 후 연결. 기존에 사용하던 메인 프로필이 감지 '플래그'를 받았을 경우 이를 분리하여 위험 분산.
- **로그인 세션 유지**: 매번 로그인하는 대신 기존 인증된 세션을 지속적으로 재사용.

## 5. 실험 결과 요약

| 실험 항목                   | 결과         | 비고                                               |
| --------------------------- | ------------ | -------------------------------------------------- |
| **기본 Playwright 연결**    | ❌ 실패      | 초기 10개 이후 로딩 안 됨, 감지 의심               |
| **네트워크 인터셉션 수정**  | ✅ 성공      | API 구조 파싱은 성공했으나 데이터 공급량 제한됨    |
| **"결과 더보기" 버튼 클릭** | ⚠️ 부분 성공 | 버튼 클릭은 수행되나 콘텐츠가 추가로 로드되지 않음 |

## 💡 결론 및 향후 계획

현재의 `Playwright + MCP` 방식은 LinkedIn의 강력한 보안 정책에 의해 콘텐츠가 필터링되고 있을 가능성이 매우 높습니다. 다음 단계로 **Stealth 플러그인 도입** 및 **비정형 스크롤 패턴**을 적용하여 감지 시스템을 우회하는 테스트를 진행할 예정입니다.
