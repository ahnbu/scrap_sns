# Plan: X UI 최적화 및 전역 데이터 정렬 정규화

X(Twitter) 통합 이후 발생하는 UI 시각적 문제와 데이터 정렬 왜곡을 해결하고, 전체 시스템의 데이터 일관성을 확보합니다.

## 1. 분석 및 문제점
- **아이콘 구식**: 현재 파란색 트위터 아이콘 및 명칭 사용 중. 최신 "X" 브랜드로 교체 필요.
- **본문 누락**: X 게시글의 본문이 웹 뷰어에서 비어 보임. (`body` vs `full_text` 매핑 이슈 추정)
- **정렬 왜곡**: 이번에 일괄 추가된 수십 개의 X 북마크가 모두 최신 순서(최상단)에 배치됨. 실제 저장/작성 날짜를 기준으로 과거 기록들 사이에 배치되어야 함.

## 2. 해결 방안
### 2.1 UI/UX 고도화
- `index.html`: 필터 칩 영역에 X 로고 추가.
- `web_viewer/script.js`: `platformConfig` 업데이트. 
    - 색상: `#000000` (또는 매우 어두운 회색)
    - 명칭: `X`
    - 아이콘: `close` (또는 X 전용 SVG)

### 2.2 렌더링 버그 수정
- `twitter_scrap.py`에서 저장한 `body` 필드가 `script.js`의 `createCard`에서 정확히 참조되는지 확인.
- 플랫폼별 특이 케이스(인용 트윗 등)로 인해 본문이 중첩된 경우의 처리 로직 보완.

### 2.3 데이터 정규화 (Sequence ID 재부여)
- **방법**: `total_scrap.py`에 일시적인 정규화 로직 적용.
- **절차**:
    1. 모든 플랫폼의 데이터를 합침.
    2. 전체를 `timestamp` (또는 `date`) 오름차순으로 정렬.
    3. `sequence_id`를 1부터 순차적으로 다시 부여.
    4. 결과를 `total_full_*.json`으로 저장.
- **효과**: 신규 유입된 과거 데이터가 타임라인의 올바른 위치에 삽입됨.
