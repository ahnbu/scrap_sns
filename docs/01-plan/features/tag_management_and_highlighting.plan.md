# [Plan] 태그 관리 및 강조 표시 기능 구현

## 1. 개요
현재 웹 뷰어의 태그들은 모두 동일한 스타일로 표시되고 있습니다. 특정 키워드나 중요한 태그를 시각적으로 강조하여 정보의 가독성을 높이고, 사용자가 직접 어떤 태그를 강조할지 설정할 수 있는 기능이 필요합니다.

## 2. 조사 결과 (Analysis)
- **현재 상태**: 
    - `web_viewer/script.js`의 `renderTags()` 함수에서 모든 태그를 동일한 스타일로 렌더링함.
    - 태그의 중요도나 타입을 구분하는 데이터 구조가 없음.
    - 설정 모달이 존재하지만, 태그 타입을 변경하는 기능은 누락됨.
- **사용자 요구사항**:
    - 중요한 태그(Primary Tag)는 강조 표시.
    - 기본 태그(Basic Tag)는 현재 스타일 유지.
    - 설정 탭에서 태그별 타입을 관리(Primary <-> Basic 토글).
    - 설정값은 로컬 스토리지에 저장되어 유지되어야 함.

## 3. 해결 방안 (Proposed Solution)
### 3.1. 시각적 스타일링 (`style.css` 수정)
- `.tag-primary` 클래스 추가: 배경색 강조(테마 컬러), 폰트 굵기 변경 등 시각적 차별화.
- `.tag-basic` 클래스 명시: 기존 스타일 유지 및 클래스화.

### 3.2. 설정 UI 확장 (`index.html`, `script.js` 수정)
- 설정 모달(`managementModal`)의 탭 네비게이션에 "태그 관리" 항목 추가.
- "태그 관리" 탭 클릭 시, 현재 데이터에 존재하는 모든 태그 목록을 렌더링하고 각 태그 옆에 '강조' 여부를 토글할 수 있는 스위치 또는 버튼 배치.

### 3.3. 데이터 관리 로직 (`script.js` 수정)
- `sns_tag_types`라는 키로 로컬 스토리지에 `{ "태그명": "primary" | "basic" }` 형태의 객체 저장.
- 페이지 로드 시 및 설정 변경 시 해당 데이터를 참조하여 스타일 적용.
- `renderTags()` 함수를 수정하여 태그 렌더링 시 타입을 판단하고 적절한 CSS 클래스 주입.

## 4. 상세 계획 (Tasks)
1. [ ] `web_viewer/style.css`: `.tag-primary` 및 `.tag-basic` 스타일 정의.
2. [ ] `web_viewer/index.html`: 설정 모달 내 탭 메뉴 및 컨텐츠 영역에 "태그 관리" 관련 마크업 추가.
3. [ ] `web_viewer/script.js`:
    - [ ] 로컬 스토리지 연동 로직 (`loadTagTypes`, `saveTagType`) 구현.
    - [ ] 태그 관리 탭 렌더링 함수 (`renderTagManagementList`) 구현.
    - [ ] 태그 렌더링 시 타입별 클래스 적용 로직 수정.
    - [ ] 탭 전환 이벤트 핸들러 업데이트.
4. [ ] 검증: 설정에서 태그 타입을 변경했을 때 실제 피드에 즉시 반영되는지 확인.

## 5. 기대 효과
- 중요한 키워드에 대한 시각적 인지 속도 향상.
- 사용자 맞춤형 뷰어 환경 제공.
