# Plan: X(Twitter) 북마크 실시간 크롤러 (Playwright)

로컬 파일 기반의 분석 도구를 넘어, 실제로 브라우저를 제어하여 최신 북마크를 수집하는 실시간 크롤러를 구현합니다.

## 1. 개요
- **기술 스택**: Python, Playwright, BeautifulSoup4
- **대상**: X(Twitter) 북마크 페이지 (`https://x.com/i/bookmarks`)
- **특징**: 브라우저 위치 제어, 세션 유지, 실시간 패킷 캡처 및 DOM 분석 결합

## 2. 실시간 크롤링 전략
### 2.1 브라우저 제어 및 인증
- `threads_scrap.py`의 설정을 계승하여 `WINDOW_X=0`으로 초기 설정.
- `auth/auth_twitter.json` 파일을 사용하여 로그인 세션을 유지하고, 필요 시 수동 로그인을 위한 대기 로직 포함.

### 2.2 하이브리드 수집 (Network + DOM)
- **네트워크 인터셉트**: 브라우저가 스크롤하면서 받아오는 모든 GraphQL 응답(JSON)을 가로채어 파싱.
- **DOM 스캔**: 초기 로딩된 페이지의 HTML을 BeautifulSoup으로 즉시 분석하여 네트워크 패킷보다 먼저 수집.

### 2.3 스마트 업데이트
- `--mode update` 시, 기존 저장된 최상단 게시글을 발견하면 자동으로 스크롤을 멈추고 수집을 종료.

## 3. 구현 단계별 계획
### 3.1 인프라 (Playwright)
- 브라우저 실행 및 컨텍스트(뷰포트, 위치) 설정.
- `dotenv`를 통한 계정 정보 연동.

### 3.2 수집 엔진
- `response` 이벤트 핸들러 구현 (JSON 데이터 확보).
- 스크롤 루프 구현 (더보기 버튼 클릭 또는 자동 스크롤).

### 3.3 통합
- 기존 `extract_from_json`, `extract_from_html` 함수를 재사용하여 코드 중복 최소화.
- `total_scrap.py` 실행 시 실시간 브라우저가 실행되도록 연동.

## 4. 고려사항
- **차단 방지**: 스크롤 간 랜덤 지연 시간(Sleep) 적용.
- **인증 만료**: 세션 만료 시 브라우저가 닫히지 않고 사용자가 재로그인할 수 있도록 대기 로직 추가.
