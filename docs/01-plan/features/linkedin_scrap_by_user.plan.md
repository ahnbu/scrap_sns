# linkedin_scrap_by_user 기획서

> **요약**: 특정 사용자의 링크드인 활동 페이지(`recent-activity/all/`)를 스크랩하여 사용자별로 데이터를 관리하는 기능을 구현합니다.
>
> **프로젝트**: scrap_sns
> **버전**: 1.0.0
> **작성자**: Gemini CLI
> **날짜**: 2026-02-07
> **상태**: 완료

---

## 1. 개요

### 1.1 목적

기존의 '저장된 게시물' 스크랩 기능을 확장하여, 특정 사용자의 공개된 활동 게시물을 스크랩할 수 있도록 합니다. 이를 통해 특정 인물이나 기업의 게시글을 추적하고 아카이빙할 수 있습니다.

### 1.2 배경

현재 `linkedin_scrap.py`는 본인의 'Saved Posts'만 스크랩 가능합니다. 특정 사용자의 콘텐츠를 분석하거나 수집하기 위해 사용자 ID 기반의 스크래핑 기능이 필요하다는 요구사항이 있었습니다.

### 1.3 관련 문서

- 요구사항: 사용자 요청 사항 (파일 이름, 저장 폴더, 기간/개수 옵션 등)
- 참고 자료: `linkedin_scrap.py` (기존 스크래핑 로직 및 데이터 구조)
