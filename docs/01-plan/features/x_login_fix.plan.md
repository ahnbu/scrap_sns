# Plan: X(Twitter) 자동화 탐지 회피 및 로그인 문제 해결

Playwright를 통한 X 로그인 시 발생하는 차단 문제를 해결하기 위해 실제 브라우저 위장 및 영구 컨텍스트 기술을 도입합니다.

## 1. 분석 결과 반영
- **에러 원인**: X의 봇 탐지 시스템이 브라우저 지문 및 자동화 플래그(`navigator.webdriver`)를 감지하여 접근 차단.
- **해결 핵심**: 
    1. 실제 Google Chrome 브라우저 사용 (`channel="chrome"`)
    2. 자동화 플래그 제거 (`AutomationControlled` 비활성화)
    3. 영구 사용자 데이터 디렉토리 사용을 통한 신뢰도 확보

## 2. 대응 방안 수립
### 2.1 브라우저 구동 방식 변경
- 기존의 임시 컨텍스트 방식에서 `launch_persistent_context` 방식으로 전환.
- 사용자 데이터 저장 경로를 `auth/x_user_data`로 지정하여 쿠키 및 캐시를 영구 보존.

### 2.2 탐지 회피 설정 적용
- `args`: `["--disable-blink-features=AutomationControlled", "--no-sandbox"]`
- `user_agent`: 최신 Chrome 버전의 User-Agent를 명시적으로 지정.

### 2.3 로그인 프로세스 최적화
- 로그인 페이지 감지 시 사용자가 직접 입력할 수 있는 대기 시간을 무제한으로 설정.
- 세션이 저장된 이후에는 별도의 로그인 과정 없이 바로 북마크 페이지로 진입하도록 구성.

## 3. 구현 단계
1. **코드 수정**: `twitter_scrap.py`의 `main()` 함수 내 브라우저 실행 로직 업데이트.
2. **테스트**: 브라우저를 띄워 로그인 수행 및 차단 메시지 발생 여부 확인.
3. **검증**: 로그인 성공 후 스크립트를 재실행했을 때 세션이 유지되는지 확인.

## 4. 기대 효과
- X의 봇 탐지 시스템 우회 및 안정적인 로그인 성공.
- 매번 로그인을 반복할 필요 없는 지속적인 데이터 수집 환경 구축.
