# linkedin_load_more_button_improvement 기획서

> **요약**: 무한 스크롤 중 나타나는 "결과 더 보기" 버튼을 감지하고 클릭하여 끊김 없는 데이터 수집이 가능하도록 개선합니다.
>
> **프로젝트**: scrap_sns
> **버전**: 1.2.0
> **작성자**: Gemini CLI
> **날짜**: 2026-02-07
> **상태**: Draft

---

## 1. 개요

### 1.1 목적

링크드인 사용자 활동 페이지에서 데이터 수집 중, 무한 스크롤이 멈추고 "결과 더 보기" 버튼이 나타나는 경우가 있습니다. 현재의 스크롤 전용 로직으로는 더 이상의 데이터를 로드할 수 없으므로, 해당 버튼을 명확히 식별하여 클릭하는 로직을 추가하여 수집 자동화의 완성도를 높입니다.

### 1.2 배경

- 링크드인은 특정 시점 이후에 무한 스크롤 대신 사용자 상호작용(버튼 클릭)을 요구함.
- 현재 코드에도 유사한 로직이 있으나, 특정 클래스(`scaffold-finite-scroll__load-button`)나 ID 패턴을 가진 버튼에 대해 더 견고한 처리가 필요함.

### 1.3 관련 문서

- 참고 자료: `linkedin_scrap_by_user.py` (현재 코드)
- 타겟 버튼 예시: `<button class="... scaffold-finite-scroll__load-button" ...> 결과 더보기 </button>`

## 3. 요구사항

### 3.1 기능적 요구사항

| ID | 요구사항 | 우선순위 | 상태 |
|----|----------|----------|------|
| FR-01 | `scaffold-finite-scroll__load-button` 클래스를 포함한 버튼 탐지 | 높음 | 대기 |
| FR-02 | 버튼 발견 시 클릭 이벤트 발생 및 데이터 로드 대기 | 높음 | 대기 |
| FR-03 | 버튼이 보이지 않을 경우 기존의 `window.scrollTo` 방식 유지 | 보통 | 대기 |
| FR-04 | "결과 더 보기" 또는 "Show more results" 텍스트 포함 여부 확인 | 보통 | 대기 |

### 3.2 비기능적 요구사항

| 카테고리 | 기준 | 측정 방법 |
|----------|------|-----------|
| 안정성 | 버튼 클릭 시 페이지 오류나 무한 대기가 발생하지 않아야 함 | 실행 로그 및 브라우저 확인 |
| 유연성 | 버튼의 ID가 변경되더라도 클래스나 텍스트로 탐지 가능해야 함 | 코드 리뷰 |

## 5. 아키텍처 고려 사항

### 5.1 주요 설계 결정

- **탐지 방식**: Playwright의 `page.locator`와 CSS Selector를 조합하여 버튼을 찾습니다. 특히 `scaffold-finite-scroll__load-button` 클래스를 우선적으로 사용합니다.
- **예외 처리**: 버튼 클릭 중 발생할 수 있는 Timeout이나 ElementNotInteractable 에러를 `try-except`로 감싸서 프로그램이 중단되지 않게 합니다.
