# linkedin_scrap_order_analysis 기획서

> **요약**: 링크드인 사용자 게시글 스크래핑 시 데이터 순서가 섞이는 원인을 분석하고, 이를 최신순으로 정렬하여 저장할 수 있는 개선 방안을 수립합니다.
>
> **프로젝트**: scrap_sns
> **버전**: 1.0.0
> **작성자**: Gemini CLI
> **날짜**: 2026-02-08
> **상태**: 진행 중

---

## 1. 개요

### 1.1 목적

현재 `linkedin_scrap_by_user.py`를 통해 수집된 데이터의 `sequence_id` 및 저장 순서가 실제 게시글 작성 시간과 일치하지 않고 섞여 나오는 현상을 진단하고, 이를 해결하기 위한 이론적 근거와 개선 방향을 제시합니다.

### 1.2 배경

- 사용자가 게시글을 시간 순으로 확인하고 싶어 하나, 수집 결과물에서 시간 순서가 뒤섞여 있음.
- 네트워크 응답 처리 방식이나 LinkedIn API의 데이터 전달 구조에 기인한 것으로 추정됨.

### 1.3 관련 문서

- `linkedin_scrap_by_user.py` (현재 수집 스크립트)
