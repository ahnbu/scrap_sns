## 1. 개요

### 1.1 목적

기존 `linkedin_scrap_by_user.py`의 편의성과 효율성을 강화합니다. 특히 이미 수집된 최신 데이터를 건너뛰는 기능과, 작업 완료 후 상세한 통계 정보를 제공하여 사용자가 스크래핑 상태를 명확히 파악할 수 있도록 합니다.

### 1.2 배경

- 사용자가 특정 기간(예: 31일~90일 전)만 타겟팅하여 수집하고 싶어함.
- 수집 완료 후 소요 시간 및 성공/실패 건수에 대한 요약 정보가 부재함.

### 1.3 관련 문서

- 참고 자료: `linkedin_scrap_by_user.py` (기존 코드)
