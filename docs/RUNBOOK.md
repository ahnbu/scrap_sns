# 운영 가이드 (RUNBOOK.md)

이 문서는 SNS Feed Viewer 서비스의 운영 및 문제 해결을 위한 지침입니다.

## 1. 서비스 실행 절차

1. **Viewer 실행**: `run_viewer.bat`을 실행하거나 `npm run view`를 호출합니다.
2. **스크래핑 시작**: 브라우저가 열리면 화면 상단의 'Run Scraper' 버튼을 클릭합니다.
3. **데이터 확인**: 수집된 데이터는 `output_threads/` 또는 `output_linkedin/` 폴더에 JSON 형태로 저장됩니다.

## 2. 서비스 종료 절차

- `stop_viewer.bat`을 실행하여 백그라운드에서 실행 중인 Flask 서버와 Python 프로세스를 정리합니다.

## 3. 주요 문제 해결 (Troubleshooting)

### 3.1 로그인 실패 또는 세션 유효하지 않음

- **원인**: `.env.local` 정보가 틀렸거나 브라우저에서 2단계 인증이 필요할 수 있습니다.
- **해결**: `.env.local`의 자격 증명을 확인하고, 스크랩 스크립트 실행 시 열리는 브라우저에서 수동 로그인을 한 번 완료하십시오.

### 3.2 403 Forbidden (Meta CDN)

- **원인**: Instagram/Threads의 이미지/동영상 링크가 외부 호출을 차단한 경우입니다.
- **해결**: 뷰어 내의 네트워크 프록시 혹은 데이터 로딩 로직(`data_loading_review.md` 참조)을 확인하십시오.

### 3.3 서버 포트 5000 충돌

- **원인**: 다른 프로세스가 5000번 포트를 사용 중입니다.
- **해결**: `stop_viewer.bat`을 실행하거나 리소스 모니터에서 해당 포트를 점유 중인 PID를 찾아 종료하십시오.

## 4. 모니터링 및 로그

- `server.py` 실행 시 콘솔에 출력되는 로그를 통해 API 요청 및 데이터 처리 상태를 모니터링할 수 있습니다.
