## Task 1: P0-1 — server.py `stderr_val` NameError 수정

**파일:**
- Modify: `server.py:151-157`

**배경:**
`subprocess.Popen`에서 `stderr=subprocess.STDOUT`으로 설정하여 stderr가 stdout에 병합되었다.
따라서 출력은 `stdout_val`에 담겨 있는데, 실패 분기(151·156행)에서 존재하지 않는 `stderr_val`을 참조한다.

**Step 1: 현재 버그 확인**

`server.py` 149~157행을 확인한다:
```python
else:
    print(f"❌ 스크래핑 실패 (코드: {process.returncode})")
    if stderr_val:                          # ← NameError
        print(f"--- Error Details ---\n{stderr_val}\n--------------------")
    return jsonify({
        "status": "error",
        "message": f"Scraping failed with exit code {process.returncode}",
        "error": stderr_val                 # ← NameError
    }), 500
```

**Step 2: 수정 — `stderr_val` → `stdout_val` 교체**

`server.py` 151~156행을 다음으로 교체한다:
```python
        else:
            print(f"❌ 스크래핑 실패 (코드: {process.returncode})")
            if stdout_val:
                print(f"--- Error Details ---\n{stdout_val}\n--------------------")
            return jsonify({
                "status": "error",
                "message": f"Scraping failed with exit code {process.returncode}",
                "error": stdout_val
            }), 500
```

**Step 3: 구문 검증**

```bash
python -m py_compile server.py && echo "OK"
```
Expected: `OK`

**Step 4: 커밋**

```bash
git add server.py
git commit -m "fix(server): stderr_val NameError → stdout_val 교체 — 스크래핑 실패 시 NameError 방지"
```

## Task 3: P0-3 — 모듈 레벨 argparse 캡슐화

**파일:**
- Modify: `thread_scrap.py:49-52` (전역 → main 함수)
- Modify: `linkedin_scrap.py:22-25` (전역 → main 함수)
- Modify: `twitter_scrap.py:38-40` (전역 → main 함수)
- Modify: `total_scrap.py:61-63` (전역 → main 함수)

**배경:**
4개 파일 모두 모듈 최상위(전역 범위)에서 `parser.parse_args()`를 호출한다.
`import`만 해도 `parse_args()`가 실행되어 테스트·외부 호출 시 `SystemExit`로 종료된다.
수정 패턴: 전역 `args` 변수와 그에 의존하는 설정을 `main()` 함수 내로 이동하고,
`if __name__ == "__main__": main()` 블록을 추가(또는 활용)한다.

### Sub-task 3-A: thread_scrap.py

**Step 1: 전역 범위 확인**

49~52행:
```python
parser = argparse.ArgumentParser(description='Threads 스크래퍼')
parser.add_argument('--mode', choices=['all', 'update'], default='all', help='크롤링 모드 (all: 전체, update: 증분)')
args = parser.parse_args()
CRAWL_MODE = "update only" if args.mode == "update" else "all"
```

**Step 2: 파일 끝 구조 확인**

파일 마지막에 `if __name__ == "__main__":` 블록이 있는지 확인한다:
```bash
grep -n "__main__" thread_scrap.py
```

**Step 3: 수정 내용**

전역 범위의 4줄을 삭제하고, 기존 실행 진입점(스크래핑을 시작하는 함수 호출부) 직전에 argparse를 이동한다.
파일 구조에 따라 두 가지 방식 중 선택:

**방식 A** — `if __name__ == "__main__":` 블록이 이미 있는 경우:
```python
# 전역에서 제거
# parser / args / CRAWL_MODE 4줄 삭제

# if __name__ == "__main__": 블록 수정
if __name__ == "__main__":
    parser = argparse.ArgumentParser(description='Threads 스크래퍼')
    parser.add_argument('--mode', choices=['all', 'update'], default='all', help='크롤링 모드')
    args = parser.parse_args()
    CRAWL_MODE = "update only" if args.mode == "update" else "all"
    # 기존 실행 코드...
```

**방식 B** — `if __name__ == "__main__":` 블록이 없는 경우:
파일 하단 실행 코드를 `main()` 함수로 감싸고 아래를 추가:
```python
def main():
    parser = argparse.ArgumentParser(description='Threads 스크래퍼')
    parser.add_argument('--mode', choices=['all', 'update'], default='all', help='크롤링 모드')
    args = parser.parse_args()
    global CRAWL_MODE
    CRAWL_MODE = "update only" if args.mode == "update" else "all"
    # 기존 실행 코드...

if __name__ == "__main__":
    main()
```
