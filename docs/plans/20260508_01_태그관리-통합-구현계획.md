# 태그 관리 통합 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `자동 태그`와 `태그 관리`를 하나의 설정 화면으로 통합하고, 기존 `keyword -> tag` 규칙을 태그별 alias/키워드로 1회 변환해 신규 게시물에 자동 태그가 붙게 만든다.

**Architecture:** 기존 `web_viewer/sns_tags.json`의 URL→태그 배열은 유지한다. 새 태그 메타데이터 파일 `web_viewer/sns_tag_catalog.json`을 추가해 태그명, primary, aliases만 저장한다. 클라이언트는 태그 카탈로그와 최신 데이터 ETag 조합이 바뀐 경우에만 `/api/auto-tag/apply`를 조용히 실행해 자동 태그를 저장한다.

**Tech Stack:** Flask `server.py`, browser JavaScript `web_viewer/script.js`, `index.html`, JSON persistence, pytest, Node-based JS unit tests.

## 설정 화면 미리보기

기존 설정 탭:

```text
숨김 관리 | 자동 태그 | 태그 관리 | 전체 크롤링
```

변경 후 설정 탭:

```text
숨김 관리 | 태그 관리 | 전체 크롤링
```

`태그 관리` 탭 미리보기:

```text
┌──────────────────────────────────────────────────────────────┐
│ 태그 관리                                                     │
│ [ 태그 검색...                            ] [+ 태그] [일괄 적용] │
├──────────────────────────────────────────────────────────────┤
│ 리서치 (77개)                                  Primary ○   [⋯] │
│ Alias:  리서치  심층리서치  research                     [+] │
├──────────────────────────────────────────────────────────────┤
│ 클로드 (599개)                                 Primary ●   [⋯] │
│ Alias:  클로드  claude  Claude Code                     [+] │
├──────────────────────────────────────────────────────────────┤
│ 디자인 (145개)                                 Primary ○   [⋯] │
│ Alias:  디자인  design                                  [+] │
└──────────────────────────────────────────────────────────────┘
```

행 단위 동작:

- 태그명 자체는 항상 자동 키워드로 포함한다.
- alias/키워드는 태그명 외 추가 매칭어다.
- 태그 사용 게시물 수는 태그명 옆에 표시한다.
- `Primary`는 기존 강조 표시 기능이다.
- `[+ 태그]`는 새 태그를 만든다.
- `[+]`는 해당 태그의 alias를 추가한다.
- `[⋯]`는 태그명 변경과 태그 삭제 메뉴다.
- `일괄 적용`은 현재 카탈로그 기준으로 모든 게시물에 다시 태그를 적용한다.
- 신규 데이터 감지 시 같은 적용 로직이 조용히 1회 실행된다.

## 영구화 Surface

영향받는 영구화 surface:

- `web_viewer/sns_tags.json`: 기존 URL→태그 배열. 자동 적용 결과가 저장된다.
- `web_viewer/sns_tag_catalog.json`: 신규 태그 카탈로그. 태그별 primary와 aliases를 저장한다.
- 브라우저 `localStorage.sns_tags`: 기존 태그 캐시.
- 브라우저 `localStorage.sns_tag_catalog`: 신규 태그 카탈로그 캐시.
- 브라우저 `localStorage.sns_auto_tag_rules`: 1회 변환 입력. 변환 후 새 코드 경로에서는 읽지 않는다.
- 브라우저 `localStorage.sns_tag_types`: 1회 변환 입력. 변환 후 새 코드 경로에서는 읽지 않는다.
- 브라우저 `localStorage.sns_tag_catalog_migrated`: 1회 변환 완료 표시.
- 브라우저 `localStorage.sns_auto_tag_last_signature`: 최신 데이터와 카탈로그 조합에 대한 자동 적용 완료 표시.

DB, 검색 인덱스, 서버 캐시 스키마 변경은 없다. `_POSTS_CACHE`는 기존 posts 캐시로 유지한다.

## 마이그레이션 판단

기존 `sns_tags.json` 데이터 마이그레이션은 불필요하다. URL→태그 배열 구조를 그대로 유지하기 때문이다.

신규 카탈로그 생성은 필요하다. 실행 위치는 브라우저 초기 로드 경로다.

1회 변환 규칙:

1. 서버 `web_viewer/sns_tag_catalog.json`이 있으면 그것을 정본으로 사용한다.
2. 서버 파일이 없거나 비어 있으면 현재 `postTags`의 고유 태그 17개로 카탈로그를 만든다.
3. `localStorage.sns_tag_types[tag] === "primary"`이면 `catalog[tag].primary = true`.
4. `localStorage.sns_tag_catalog_migrated !== "true"`이면, 서버 카탈로그가 이미 있어도 `localStorage.sns_auto_tag_rules`의 `{ keyword, tag }`를 `catalog[tag].aliases`에 병합한다.
5. `keyword`가 `tag`와 같으면 aliases에는 중복 저장하지 않는다. 태그명 자체는 항상 자동 키워드로 취급한다.
6. 변환 후 `localStorage.sns_tag_catalog_migrated = "true"`를 저장하고 변경이 있으면 `/api/save-tag-catalog`로 서버에 저장한다.
7. 기존 `sns_auto_tag_rules`와 `sns_tag_types`는 삭제하지 않는다. 새 코드 경로에서 사용하지 않는다.

```javascript
function normalizeTagCatalog(rawCatalog) {
    const catalog = {};
    Object.entries(rawCatalog || {}).forEach(([tag, meta]) => {
        const name = String(tag || '').trim();
        if (!name) return;

        const aliases = Array.from(new Set((meta?.aliases || [])
            .map((alias) => String(alias || '').trim())
            .filter((alias) => alias && alias !== name)));

        catalog[name] = {
            primary: Boolean(meta?.primary),
            aliases,
        };
    });
    return catalog;
}

function buildTagCatalogFromExistingState(postTagsMap, tagTypesMap, legacyRules) {
    const catalog = {};

    Object.values(postTagsMap || {}).forEach((tags) => {
        (tags || []).forEach((tag) => {
            const name = String(tag || '').trim();
            if (!name) return;
            if (!catalog[name]) {
                catalog[name] = { primary: false, aliases: [] };
            }
        });
    });

    Object.entries(tagTypesMap || {}).forEach(([tag, type]) => {
        const name = String(tag || '').trim();
        if (!name || !catalog[name]) return;
        catalog[name].primary = type === 'primary';
    });

    (legacyRules || []).forEach((rule) => {
        const tag = String(rule?.tag || '').trim();
        const keyword = String(rule?.keyword || '').trim();
        if (!tag || !keyword || keyword === tag) return;
        if (!catalog[tag]) {
            catalog[tag] = { primary: false, aliases: [] };
        }
        if (!catalog[tag].aliases.includes(keyword)) {
            catalog[tag].aliases.push(keyword);
        }
    });

    return normalizeTagCatalog(catalog);
}
```
