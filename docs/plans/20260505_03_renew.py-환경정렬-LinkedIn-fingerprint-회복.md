# renew.py 환경 정렬 — LinkedIn fingerprint 회복 plan

## 목적

본 세션 v3/v4/v5 PoC가 확정한 결론에 따라, `scripts/auth_runtime/renew.py:renew_storage_state()` 본체에 user_agent / locale / viewport를 추가해 갱신기와 수집기의 fingerprint를 정렬한다.

이로써 LinkedIn 저장글 자동 수집(`linkedin_scrap.py --mode update`)이 다시 동작하도록 만든다.

## 배경 (요약)

- 2026-04-18 `b82d153` AUTH_HOME 중앙화 시점에 과거 `renew_auth.py`의 `Chrome/121` user_agent 명시가 신규 `scripts/auth_runtime/renew.py`로 이전되면서 누락됨.
- 2026-05-05 `855e4bf`로 `linkedin_scrap.py`가 `Chrome/122` user_agent를 명시 — 갱신기와 수집기의 fingerprint 불일치가 더 커짐.
- LinkedIn은 같은 storage_state 쿠키가 갱신과 사용 사이에 다른 fingerprint로 보이면 saved-posts를 login으로 redirect.
- v3/v4/v5에서 갱신기 환경(Chrome/122 + ko-KR + 1280x1000)으로 새 storage_state를 만들면 saved-posts 통과 확정.

- A는 함수 본체 3줄 추가, B는 함수 분리 + main 라우팅 변경 = 더 복잡

## 변경 사항

### 1. `scripts/auth_runtime/renew.py` 수정

**위치**: `scripts/auth_runtime/renew.py:24-37` (`renew_storage_state` 함수)

**현재 코드**:

```python
def renew_storage_state(name: str, url: str, target: Path) -> None:
    _require_tty()
    target.parent.mkdir(parents=True, exist_ok=True)

    with sync_playwright() as playwright:
        browser = playwright.chromium.launch(headless=False)
        context_args = {}
        if target.exists() and target.stat().st_size > 10:
            context_args["storage_state"] = str(target)
        context = browser.new_context(**context_args)
        page = context.new_page()
        page.goto(url)
        input(f"[{name}] 로그인 완료 후 Enter: ")
        context.storage_state(path=str(target))
        browser.close()
```

**변경 후**:

```python
LINKEDIN_USER_AGENT = (

def renew_storage_state(name: str, url: str, target: Path) -> None:
    _require_tty()
    target.parent.mkdir(parents=True, exist_ok=True)

    with sync_playwright() as playwright:
        browser = playwright.chromium.launch(headless=False)
        context_args = {
            "user_agent": LINKEDIN_USER_AGENT,
            "locale": "ko-KR",
            "viewport": {"width": 1280, "height": 1000},
        }
        if target.exists() and target.stat().st_size > 10:
            context_args["storage_state"] = str(target)
        context = browser.new_context(**context_args)
        page = context.new_page()
        page.goto(url)
        input(f"[{name}] 로그인 완료 후 Enter: ")
        context.storage_state(path=str(target))
        browser.close()
```

**변경 라인 수**: 6줄 추가 (상수 5줄 + context_args 3줄 → 기존 1줄 대체).

**주의: `LINKEDIN_USER_AGENT` 변수명은 모든 플랫폼이 공유하므로 `BROWSER_USER_AGENT`로 바꾸는 것도 합리적이다. 하지만 의도가 명확하도록 변수명은 그대로 유지하고 docstring/주석에 "LinkedIn fingerprint 회복 목적, 다른 플랫폼에도 동일 적용"을 명시한다.**
