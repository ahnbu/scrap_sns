# 범위 외 회귀 9건 구현 및 검증 기록

## 발단: 사용자 요청

- 사용자는 합의된 구현 계획을 실행하라고 요청했다.
- 대상은 범위 외로 남아 있던 9건의 실패 묶음이었다: 루트 뷰어 진입점 불일치, Playwright smoke/e2e 혼합 실행 충돌, Threads/LinkedIn parser 계약 드리프트.
- 구현 후 전체 테스트가 실제로 닫히는지 확인하고, 마지막에 문서 저장과 커밋까지 연결하는 것이 목표였다.

## 작업 상세내역

1. 서버 진입점과 정적 파일 서빙 계약을 복구했다.
   `server.py`가 더 이상 존재하지 않는 `web_viewer/index.html`을 찾고 있었기 때문에, 루트 `index.html`을 `/`의 정본으로 고정했다. 동시에 `web_viewer/*`와 `favicon.*`만 공개하고, 그 외 경로는 SPA fallback 전에 차단하도록 정리했다.

2. 라우팅 회귀를 고정하는 테스트를 추가했다.
   `tests/integration/test_server_routes.py`를 추가해 `/`가 루트 viewer shell을 돌려주는지, `/web_viewer/script.js`가 공개 자산인지, `/README.md` 같은 비공개 파일이 노출되지 않는지를 검증하도록 했다.

3. 뷰어를 same-origin API 호출로 정리했다.
   `web_viewer/script.js`가 `http://localhost:5000/...`를 하드코딩하고 있어 테스트와 런타임이 외부 서버 상태에 묶여 있었다. 이를 `/api/...` 형태로 바꿔 현재 origin 기준으로 동작하게 만들었다.

4. e2e 테스트를 외부 수동 서버 의존에서 분리했다.
   `tests/e2e/test_core_ui.py`는 세션 스코프 임시 Flask 서버를 직접 띄우고 종료하도록 바꿨다. 이 변경으로 로컬에 떠 있는 다른 서버 상태, 포트 점유, stale process에 흔들리지 않고 테스트가 self-contained하게 실행된다.

5. smoke 테스트 실행 모델을 하나로 통일했다.
   `tests/smoke/test_linkedin_smoke.py`, `tests/smoke/test_threads_smoke.py`, `tests/smoke/test_twitter_smoke.py`에서 `sync_playwright()` 직접 호출을 제거하고 `pytest-playwright`의 `Playwright` fixture를 사용하도록 바꿨다. 이로써 e2e와 smoke를 같은 pytest 프로세스에서 섞어도 event loop 충돌이 나지 않게 했다.

6. parser 실제 호출부와 테스트 기대값을 표준 스키마에 맞췄다.
   `thread_scrap.py`는 `clean_text(raw_text, platform='threads', username=username)`로 수정해 Threads username 제거 로직이 실제로 적용되게 했다. `utils/linkedin_parser.py`는 `clean_text(text, platform="linkedin")`으로 수정해 LinkedIn 전용 `…더보기` 제거가 동작하게 했다.

7. parser 테스트 계약도 현재 정본으로 정렬했다.
   `tests/unit/test_parsers.py`는 keyword 기반 `clean_text()` 호출을 기준으로 검증하게 바꿨고, `tests/integration/test_parser_integration.py`는 legacy `user` 대신 `username` 중심 표준 출력과 `user` 부재를 검증하게 바꿨다.

8. 전체 테스트 과정에서 추가로 드러난 보안 회귀를 같은 축에서 닫았다.
   전체 `pytest tests/ -v` 실행 중 `test_s3_url_encoded_traversal`이 200을 반환하는 문제가 새로 드러났다. `server.py`에 `..` 세그먼트 탐지를 추가해 URL-encoded traversal도 SPA fallback 전에 403으로 막았다.

- 결정: 루트 HTML 진입점, same-origin API, pytest-playwright 단일 런타임, `username` 정본 계약을 기준으로 구현했다.
- 근거: 실제 실패 재현과 현재 문서 정본이 모두 이 방향을 지지했고, 전체 `pytest tests/ -v`까지 한 번에 닫히는 구성이었다.
- 트레이드오프: 테스트 고립성과 계약 일관성은 올라갔지만, smoke/e2e가 기존보다 명시적으로 테스트 인프라를 소유하게 됐다.

## 리스크 및 미해결 이슈

- 작업 트리는 이번 변경과 무관한 사용자 변경이 이미 섞여 있었다. `BACKLOG.md`, `output_total/total_full_20260417.json`, `scrap_failures_threads.json`, `web_viewer/data.js`, `web_viewer/sns_tags.json`은 그대로 두었다.
- 테스트 실행 중 `tests/fixtures/snapshots/*`에 snapshot churn이 발생했다. 이는 검증 산출물이지만 이번 구현 논리와 직접 관련된 소스 변경은 아니므로 커밋 범위 판단 시 분리 검토가 필요하다.
- parser 호출 버그는 수정됐지만, 실제 persisted 데이터 재생성은 수행하지 않았다. 다만 이번 변경은 표준 스키마 자체를 바꾸지 않았고, 저장 surface 마이그레이션이 필요한 변경은 아니라고 판단했다.

## 다음 액션

- 이번 구현 커밋에서는 범위 외 사용자 변경과 snapshot churn을 분리해 staging 범위를 엄격히 제한한다.
- 이후 필요하면 snapshot 정책을 정리해 smoke 실행이 fixture 산출물을 갱신하지 않도록 별도 규칙을 두는 것이 좋다.
