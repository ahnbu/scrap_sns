## 의사결정 기록

### XSS 수정 방향 (C4)

- **검토 옵션**: (A) innerHTML → DOM API 전체 재작성, (B) escapeHtml + inline handler 제거
- **결정**: (B) escape + inline handler 제거
- **근거**: createCard()가 약 400줄이며 전체 재작성은 회귀 리스크가 높음. escape 적용으로 XSS 차단 효과는 동등하면서 변경 범위를 최소화
- **트레이드오프**: innerHTML 자체는 남아있어 향후 새 동적 데이터 삽입 시 escape 누락 가능성. 코드 리뷰로 보완 필요

### bare except 교체 방식 (H3)

- **결정**: `except Exception:` (변수 불필요 시) 또는 `except Exception as e:` (에러 출력 시)
- **근거**: 기존 코드 패턴(print/pass)을 유지하면서 `KeyboardInterrupt`/`SystemExit` 통과만 보장
